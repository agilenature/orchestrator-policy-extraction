  <action>
    **gold-standard-label.schema.json:**
    Create a JSON Schema defining the structure for human-verified labels:
    ```json
    {
      "type": "object",
      "required": ["episode_id", "verified_mode", "verified_reaction_label"],
      "properties": {
        "episode_id": {"type": "string"},
        "verified_mode": {"type": "string", "enum": ["Explore", "Plan", "Implement", "Verify", "Integrate", "Triage", "Refactor"]},
        "verified_reaction_label": {"type": "string", "enum": ["approve", "correct", "redirect", "block", "question", "unknown"]},
        "verified_reaction_confidence": {"type": "number", "minimum": 0, "maximum": 1},
        "constraint_should_extract": {"type": "boolean", "description": "Whether this episode should have produced a constraint"},
        "notes": {"type": "string"},
        "reviewer": {"type": "string"}
      }
    }
    ```

    **gold_standard.py:**
    - `export_for_review(conn: duckdb.DuckDBPyConnection, output_dir: Path, sample_size: int = 100) -> int`:
      - Queries episodes table: `SELECT episode_id, session_id, mode, risk, reaction_label, reaction_confidence, outcome_type, observation, orchestrator_action, outcome, provenance FROM episodes`
      - Stratified sampling: ensure minimum 5 examples per mode and per reaction_label where available, fill remaining with random selection up to sample_size
      - For each episode, writes a JSON file at `output_dir/episodes/{episode_id}.json` containing the full episode data
      - For each episode, writes a template label file at `output_dir/labels/{episode_id}.json` containing: episode_id (pre-filled), verified_mode (blank string), verified_reaction_label (blank string), verified_reaction_confidence (null), constraint_should_extract (null), notes (blank string), reviewer (blank string)
      - Returns count of exported episodes
    - `import_labels(label_dir: Path, schema_path: Path) -> tuple[list[dict], list[str]]`:
      - Reads all .json files from label_dir
      - Validates each against gold-standard-label.schema.json
      - Returns (valid_labels, error_messages)
      - Skips incomplete labels (where verified_mode or verified_reaction_label is blank/empty string)

    **tests/test_gold_standard.py:**
    - Test export_for_review with mock DuckDB containing sample episodes (verify files created, template labels have correct structure)
    - Test stratified sampling ensures coverage across modes and reaction labels
    - Test import_labels with valid and invalid label files
    - Test import_labels skips incomplete labels (blank verified_mode or verified_reaction_label)
    - Test import_labels validates against JSON Schema (rejects malformed files)
    - At least 8 tests total

<task type="auto">
  <name>Task 2: Quality metrics calculator</name>
  <files>
    src/pipeline/validation/metrics.py
    tests/test_metrics.py
  </files>
  <action>
    **metrics.py:**
    - `MetricsReport` dataclass with fields: mode_accuracy (float|None), reaction_accuracy (float|None), reaction_avg_confidence (float|None), constraint_extraction_rate (float|None), per_mode_accuracy (dict[str, float|None]), per_reaction_accuracy (dict[str, float|None]), sample_size (int), thresholds_met (dict[str, bool])
    - `compute_metrics(gold_labels: list[dict], conn: duckdb.DuckDBPyConnection, constraints: list[dict] | None = None) -> MetricsReport`:
      - Reads pipeline episodes by episode_id from DuckDB
      - **Mode accuracy:** Compares pipeline mode vs verified_mode. Count matches / total compared. Per-mode breakdown.
      - **Reaction accuracy:** Compares pipeline reaction_label vs verified_reaction_label. Count matches / total compared. Per-label breakdown.
      - **Reaction avg confidence:** Computes average reaction_confidence for episodes where gold label matches pipeline label.
      - **Constraint extraction rate:** For episodes where gold label has `constraint_should_extract=True`, check whether a constraint was actually extracted for that episode. Linkage method: Load the constraints list (passed as `constraints` parameter, typically from `ConstraintStore.constraints`). Each constraint has an `examples` array of `{episode_id, violation_description}` dicts (see `ConstraintExtractor.extract()` line 121-126). Build a set of all episode_ids across all constraints' examples arrays. For each gold label with `constraint_should_extract=True`, check if that episode_id appears in the set. Rate = episodes with constraint found / total constraint_should_extract=True episodes.
      - **Zero-denominator handling:** Returns None for any metric where the denominator is 0 (e.g., no episodes to compare, no constraint_should_extract=True labels). Do NOT raise or crash.
      - **Threshold checking:** Populate thresholds_met dict: `{"mode_accuracy": value >= 0.85, "reaction_avg_confidence": value >= 0.80, "constraint_extraction_rate": value >= 0.90}`. If metric is None, threshold is False.
    - `format_report(report: MetricsReport) -> str`: Human-readable text report with PASS/FAIL indicators per threshold.

    **tests/test_metrics.py:**
    - Test compute_metrics with all labels matching (100% accuracy)
    - Test compute_metrics with mixed matches (partial accuracy)
    - Test compute_metrics with zero denominator for each metric type (returns None, no crash)
    - Test constraint_extraction_rate with constraints list containing matching episode_ids in examples array
    - Test constraint_extraction_rate when constraint was enriched with multiple examples (ConstraintStore dedup adds to examples)
    - Test threshold checking (above, below, and None cases)
    - Test per_mode_accuracy and per_reaction_accuracy breakdowns
    - Test format_report produces readable output with PASS/FAIL
    - At least 10 tests total

    **exporter.py:**
    - `export_parquet(conn: duckdb.DuckDBPyConnection, output_path: str | Path, query: str | None = None) -> int`:
      - Default query: `SELECT * FROM episodes WHERE reaction_label IS NOT NULL`
      - Uses DuckDB native: `conn.execute(f"COPY ({query}) TO '{output_path}' (FORMAT PARQUET)")`
      - Creates parent directory if needed
      - Returns row count exported (via a COUNT query before COPY)
    - `export_parquet_partitioned(conn: duckdb.DuckDBPyConnection, output_dir: str | Path, partition_by: str = "mode") -> int`:
      - Uses: `COPY episodes TO '{output_dir}' (FORMAT PARQUET, PARTITION_BY ({partition_by}))`
      - Returns total row count

    **validate.py (new CLI module):**
    Create a Click group with three subcommands:

    `validate export`:
    - Options: --db (DuckDB path), --output-dir (default: data/gold-standard), --sample-size (default: 100)
    - Calls export_for_review()
    - Prints count of exported episodes and paths

    `validate metrics`:
    - Options: --db (DuckDB path), --labels-dir (default: data/gold-standard/labels), --constraints (default: data/constraints.json)
    - Calls import_labels() then compute_metrics()
    - Prints formatted MetricsReport with pass/fail indicators for thresholds
    - Saves report JSON to data/gold-standard/metrics/report.json

    `validate export-parquet`:
    - Options: --db (DuckDB path), --output (default: data/export/episodes.parquet), --partition-by (optional)
    - Calls export_parquet() or export_parquet_partitioned()
    - Prints count and output path

    **CLI wiring in __main__.py (currently imports extract.main() directly -- must refactor):**
    The current `__main__.py` is:
    ```python
    from src.pipeline.cli.extract import main
    if __name__ == "__main__":
        main()
    ```
    Refactor to use a Click group structure:
    1. Create a top-level `@click.group()` called `cli` in `__main__.py`
    2. Add the existing extract command as a subcommand: `cli.add_command(main, name="extract")`  (importing `main` from `src.pipeline.cli.extract`)
    3. Add the validate group as a subcommand: `cli.add_command(validate_group, name="validate")` (importing `validate_group` from `src.pipeline.cli.validate`)
    4. The `if __name__ == "__main__": cli()` entry point
    5. Usage changes from `python -m src.pipeline.cli.extract <path>` to `python -m src.pipeline.cli extract <path>` and `python -m src.pipeline.cli validate export [options]`
    6. Do NOT modify extract.py itself -- the existing `@click.command()` decorator is compatible with `add_command()`.
