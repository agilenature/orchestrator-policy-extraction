## Decisions Made
- **VerdictCollector.input_fn injectable:** Enables test isolation without subprocess calls or monkeypatching stdin. Click's CliRunner redirects `input()` to provided stdin, so the default `input_fn=input` works in both production and test contexts.
- **--constraints CLI option added:** The PoolBuilder reads constraints from `data/constraints.json` by default, which would leak real data into tests. Added `--constraints` option to enable test isolation.
- **session_id left as None:** No session tracking infrastructure exists yet. Extension point for Phase 14+ when session IDs are available in live governance.
- **Append-only contract at API level:** ReviewWriter has no `update()` or `delete()` methods. The UNIQUE constraint on `identification_instance_id` enforces at-most-once semantics at the DB level.
