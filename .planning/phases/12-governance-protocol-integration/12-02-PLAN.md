2. **Create `src/pipeline/governance/parser.py`:**
   Header-hierarchy Markdown parser using Python `re` stdlib (per locked decision 1).

   Classes/functions:
   - `ParsedEntity` dataclass: `entity_type: str` (dead_end, assumption, scope_decision, method_decision), `title: str`, `content: str`, `source_section: str`
   - `ParsedSection` dataclass: `header_level: int`, `header_text: str`, `section_type: str`, `body: str`
   - `GovDocParser` class with:
     - Class-level `SECTION_KEYWORDS` dict mapping section_type -> list of keyword phrases:
       - `"failure_story"`: `["failure stories", "ways we could fail", "dead ends"]`
       - `"assumption"`: `["assumptions", "key assumptions", "core assumptions"]`
       - `"scope_decision"`: `["scope decisions"]`
       - `"method_decision"`: `["method decisions"]`
       NOTE: "Decisions" alone should NOT match -- per locked decision 12, the header must specifically say "Scope Decisions" or "Method Decisions". A bare "Decisions" header is ambiguous and should be skipped.
     - `_HEADER_RE = re.compile(r'^(#{2,3})\s+(.+)$', re.MULTILINE)`
     - `_LIST_ITEM_RE = re.compile(r'^[-*]\s+(.+)', re.MULTILINE)` for top-level bullets
     - `_NUMBERED_ITEM_RE = re.compile(r'^\d+\.\s+(.+)', re.MULTILINE)` for numbered items
     - `parse_document(content: str, source_id: str = "") -> list[ParsedEntity]`:
       1. Find all H2/H3 headers via regex
       2. For each header, classify via `_classify_header(text)` using case-insensitive keyword substring matching
       3. Extract body text between this header and the next header of same or higher level
       4. Within classified sections:
          - If section has H3 sub-headings: each H3 block = one entity (H3 title = entity title, body under H3 = entity content). Aggregate continuation lines.
          - If section has top-level list items (but no H3s): each list item = one entity. For multi-line list items, continuation lines (indented) are part of the same item. Only parse at base indentation (no nested sub-items as separate entities).
          - Entity type mapping: failure_story -> "dead_end", assumption -> "assumption" (temporary -- ingestor maps to constraint), scope_decision -> "scope_decision", method_decision -> "method_decision"
       5. Return list of ParsedEntity
     - `_classify_header(text: str) -> str | None`: Lowercase text, check each keyword list with `any(kw in lower for kw in keywords)`, return section_type or None

   IMPORTANT per locked decision 2: Classification granularity is top-level list items = one entity each. H3 sub-heading blocks also work (content aggregated under H3 = one entity). Linguistic regex is secondary check only -- do NOT use it for entity splitting.

3. **Create `src/pipeline/governance/ingestor.py`:**
   Dual-store ingestor orchestrating sequential writes (per locked decision 7).

   Classes:
   - `GovIngestResult` Pydantic model: `constraints_added: int`, `constraints_skipped: int`, `wisdom_added: int`, `wisdom_updated: int`, `wisdom_skipped: int`, `errors: list[str]`, `total_entities: int` (computed property), `is_bulk: bool` (computed property using threshold)
   - `GovDocIngestor` class:
     - `__init__(constraint_store: ConstraintStore, wisdom_store: WisdomStore, bulk_threshold: int = 5)`
     - `_FORBIDDEN_RE = re.compile(r'\b(must not|never|forbidden|do not|shall not)\b', re.IGNORECASE)` (per locked decision 8)
     - `ingest_file(path: Path, source_id: str | None = None, dry_run: bool = False) -> GovIngestResult`:
       1. Read file content
       2. Parse via `GovDocParser().parse_document(content, source_id or path.stem)`
       3. If zero entities parsed, return result with warning in errors
       4. Separate entities by type: assumptions -> constraints, failure_story/dead_end -> wisdom, scope_decision/method_decision -> wisdom only (per locked decision 12: DECISIONS.md produces only wisdom, NOT constraints)
       5. If dry_run: return result with counts but do not write
       6. **Sequential write per locked decision 7:**
          a. Write constraints FIRST: For each assumption entity:
             - Build constraint dict matching constraint.schema.json:
               - `constraint_id`: SHA-256(text + JSON(scope_paths))[:16] per existing pattern
               - `text`: entity content
               - `severity`: `_determine_severity(text)` -- default "requires_approval", upgraded to "forbidden" if `_FORBIDDEN_RE` matches (per locked decision 8)
               - `scope`: `{"paths": []}` (repo-wide for governance-ingested)
               - `detection_hints`: `[entity.content[:80]]`
               - `source_episode_id`: ""
               - `created_at`: current UTC ISO
               - `examples`: `[]`
               - `type`: "behavioral_constraint"
               - `status`: "active"
               - `source`: "govern_ingest"
               - `status_history`: `[{"status": "active", "changed_at": <created_at>}]`
               - `bypassed_constraint_id`: None
               - `supersedes`: None
               - `source_excerpt`: entity.content (the full original text)
             - Call `constraint_store.add(constraint_dict)` -- returns True/False
             - Track added/skipped counts
          b. Call `constraint_store.save()` to persist (CRITICAL per pitfall 2)
          c. Collect all constraint_ids from this document for co-occurrence linkage
          d. Write wisdom SECOND: For each dead_end/scope_decision/method_decision entity:
             - Build WisdomEntity via `WisdomEntity.create()`:
               - `entity_type`: from parsed entity type
               - `title`: entity title
               - `description`: entity content
               - `context_tags`: ["governance", "pre-mortem"] for dead_end, ["governance", "decisions"] for scope/method
               - `scope_paths`: []
               - `confidence`: 1.0
               - `source_document`: source_id
               - `source_phase`: 12
               - `metadata`: `{"related_constraint_ids": list_of_constraint_ids}` (per locked decision 9: co-occurrence heuristic -- ALL constraints from same document linked to ALL dead_ends from same document)
             - Call `wisdom_store.upsert(entity)`
             - Track added/updated counts (check if entity existed via `wisdom_store.get()` first)
       7. Return GovIngestResult with all counts
     - `_determine_severity(text: str) -> str`: Returns "forbidden" if _FORBIDDEN_RE matches, else "requires_approval"
