  <action>
    **Create `src/pipeline/runner.py`** with:

    `PipelineRunner`:
    - `__init__(self, config: PipelineConfig, db_path: str = "data/ope.db")`:
      - Store config
      - Create DuckDB connection via get_connection(db_path)
      - Create schema via create_schema(conn)
      - Initialize EventTagger(config) and EpisodeSegmenter(config)
      - Compute config_hash (SHA-256 of config YAML content, first 8 chars) for provenance

    - `run_session(self, jsonl_path: str | Path, repo_path: str | Path | None = None) -> dict`:
      Main pipeline method. Steps:

      1. **Load JSONL** via DuckDB: Call `load_jsonl_to_duckdb(conn, jsonl_path, session_id)` where session_id is derived from the filename (UUID before .jsonl extension)
      2. **Normalize JSONL events**: Call `normalize_jsonl_events(conn, session_id)` to get list[CanonicalEvent]
      3. **Load git history** (optional): If repo_path provided, call `parse_git_history(repo_path)` to get git events
      4. **Merge + normalize**: Call `normalize_events(jsonl_events, git_events, config)` for temporal alignment and deduplication
      5. **Validate event quality** (Q16 locked decision): Count events that failed normalization. If invalid_rate > config.validation.invalid_event_abort_threshold (default 10%), abort this session with error report. Use config.validation.abort_scope='per_session' -- abort this session, continue to next.
      6. **Tag events**: Call `tagger.tag(canonical_events)` to get list[TaggedEvent]
      7. **Segment events**: Call `segmenter.segment(tagged_events)` to get list[EpisodeSegment]. Attach config_hash to each segment.
      8. **Write to DuckDB**: Call `write_events(conn, canonical_events, tagged_events)` and `write_segments(conn, segments)`
      9. **Compute and return stats**: Event count, tag distribution, episode count, outcome distribution, orphan count, duplicate count, invalid count

      Return dict with: session_id, event_count, tag_distribution, episode_count, outcome_distribution, errors, warnings

    - `run_batch(self, jsonl_dir: str | Path, repo_path: str | Path | None = None) -> dict`:
      Process all .jsonl files in a directory. Use tqdm for progress. Collect per-session results. Return aggregate stats.

    - `close(self)`: Close DuckDB connection.

    Use loguru for structured logging throughout. Log each stage's input/output counts. Log timing for performance profiling.

    **Error handling architecture:**
    - Level 1 (Reject): Non-parseable JSONL lines -> skip, count as invalid
    - Level 2 (Degrade): Missing optional fields -> use defaults, log WARNING
    - Level 3 (Alternative): Failed temporal alignment -> use timestamp order with confidence=0.0
    - Level 4 (Logging): All errors logged with event context via loguru
    - Level 5 (Metrics): Stats dict includes error counts and quality metrics
    - Abort: If invalid_rate > 10% for a session, skip session and log ERROR with details
  </action>
