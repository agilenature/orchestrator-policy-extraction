# Phase 16.1 Plan 02: Conjunctive Flame Detector + Edge Generator Summary

**One-liner:** Conjunctive AND trigger (Level >= 5, Delta >= 2 above rolling median, both axes active) feeding EdgeRecord candidates via C(N,2) pair generation.

## What Was Built

### ConjunctiveFlameDetector (`src/pipeline/ddf/topology/detector.py`)

Stateful per-session detector that enforces the conjunctive trigger rule. All three conditions must be simultaneously true for a flame event to qualify for edge generation:

1. **Level threshold:** `marker_level >= 5` (absolute, not relative)
2. **Abstraction delta:** `marker_level - baseline >= 2.0` where baseline is the rolling median of the last 10 marker_levels for the session, computed from events PRIOR to the current one
3. **Co-activation:** At least 2 distinct CCD axes active in the same episode

Returns a `ConjunctiveTrigger` dataclass containing the qualifying event, computed baseline, delta, active axes, episode_id, and session_id.

### EdgeGenerator (`src/pipeline/ddf/topology/generator.py`)

Stateless transformer that converts `ConjunctiveTrigger` detections into `EdgeRecord` candidates. For N active axes, generates C(N,2) edge records (all unique sorted pairs). Each record carries:

- **evidence:** `{session_id, episode_id, flame_event_ids}` -- ground-truth pointers
- **activation_condition:** `{goal_type, scope_prefix, min_axes_simultaneously_active}` -- derived from triggering context
- **status:** Always `"candidate"` -- human review required to promote to `"active"`
- **abstraction_level:** From the triggering flame event's marker_level

### Test Suite (`tests/test_topology_detector.py`)

15 tests organized in two classes:

**Detector tests (10):**
- Conjunctive fires correctly (level 5, delta 3, 2 axes)
- Rejects level 4 (level too low despite high delta)
- Rejects level 5 with low delta (baseline too close)
- Rejects single-axis insights
- Rolling median baseline computation verified
- Empty session baseline = 0.0
- Current event excluded from own baseline
- Boundary case: exactly level 5, exactly delta 2.0 (>= not >)
- Exactly 2 axes minimum passes
- reset_session clears history

**Generator tests (5):**
- Produces EdgeRecord with all fields populated
- Deterministic edge_id (same input = same ID)
- 2 axes = 1 EdgeRecord
- 3 axes = 3 EdgeRecords (C(3,2))
- Custom relationship_text overrides auto-generated

## Architecture Notes

The detector + generator pair implements the deposit-not-detect principle: the detector identifies qualifying flame events, and the generator immediately produces EdgeRecord candidates with full evidence grounding. No intermediate "detection report" -- the output is always a deposit-ready artifact.

The baseline computation (rolling median of last 10) creates session-specific context: a Level 5 event in a session with baseline 4 (delta 1) is filtered out, while the same Level 5 in a session with baseline 2 (delta 3) qualifies. This prevents high-baseline sessions from generating noise edges.
