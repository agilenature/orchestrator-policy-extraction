**epistemological.py** -- `classify_epistemological_origin(episode: dict) -> tuple[str, float]`:

Per CONTEXT.md Q5: hard enum + confidence float. Default: 'principled'.

Classification logic:
- **reactive**: Episode has reaction_label in ('block', 'correct') AND episode mode != 'ESCALATE'
  - The constraint arose from a specific correction (stimulus-response)
  - Confidence: 0.9 if reaction is 'block', 0.8 if 'correct'
- **principled**: Episode has observation.context.constraints_in_force with 1+ entries OR episode mode is 'SUPERVISED'
  - The constraint was stated proactively, not as a reaction to an error
  - Confidence: 0.7 (harder to verify causation)
- **inductive**: Episode has 3+ examples in constraint OR constraint has detection_hints matching 3+ distinct events
  - The constraint was derived from a pattern across multiple instances
  - Confidence: 0.6 (lowest -- induction has more uncertainty)
- **Default**: 'principled' with confidence 1.0 (per locked decision)

The function returns (origin: str, confidence: float).

**generalization.py** -- Per CONTEXT.md Q6: count-based proxy.

`compute_generalization_radius(conn, constraint_id: str) -> ConstraintMetric`:
- SQL: COUNT(DISTINCT scope_path_prefix) FROM session_constraint_eval WHERE constraint_id = X
  - scope_path_prefix: first path component from evidence_json scope_path field
  - If scope_path is empty or null, use 'root' as prefix
- Also compute firing_count: COUNT(*) for this constraint_id
- Stagnation: is_stagnant = True if radius == 1 AND firing_count >= config.ddf.stagnation_min_firing_count (default 10)
- Return ConstraintMetric(constraint_id, radius, firing_count, is_stagnant)

`compute_all_metrics(conn, config: PipelineConfig) -> list[ConstraintMetric]`:
- Compute for ALL constraint_ids in session_constraint_eval
- Uses single SQL query with GROUP BY for efficiency
- Return list of ConstraintMetric

`write_constraint_metrics(conn, metrics: list[ConstraintMetric]) -> int`:
- INSERT OR REPLACE INTO constraint_metrics for each metric
- Return count written

`detect_stagnation(conn, config: PipelineConfig) -> list[ConstraintMetric]`:
- Returns only metrics where is_stagnant = True
- These are floating abstractions: constraints that fire repeatedly but only in one context

**spiral.py** -- Per CONTEXT.md DDF-06: spiral tracking.

`detect_spirals(conn, session_id: str | None = None) -> list[dict]`:
- Query session_constraint_eval joined with constraints
- Find constraints where scope_path_prefix diversity has INCREASED over time (ascending)
- A spiral is: constraint evaluated in session S1 with scope paths [A], then S2 with [A, B], then S3 with [A, B, C]
  - scope_path count is monotonically non-decreasing across sessions ordered by eval_ts
- Return list of dicts: {constraint_id, scope_path_history: list[set], spiral_length, current_radius}

`compute_spiral_depth(conn, session_id: str) -> int`:
- For a given session, find the maximum number of consecutive ascending marker_levels in flame_events
- Uses DuckDB window function: LAG(marker_level) OVER (PARTITION BY session_id ORDER BY created_at)
- Count longest streak where each level > previous level
- Return the length of the longest ascending streak

`get_spiral_promotion_candidates(conn, min_spiral_length: int = 3) -> list[str]`:
- Return constraint_ids with spiral_length >= min_spiral_length
- These are candidates for promotion to project_wisdom
