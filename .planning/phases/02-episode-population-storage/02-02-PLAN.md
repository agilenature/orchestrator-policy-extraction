<feature>
  <name>EpisodePopulator</name>
  <files>src/pipeline/populator.py, tests/test_populator.py</files>
  <behavior>
    The EpisodePopulator takes three inputs:
    1. An EpisodeSegment (from Phase 1 segmenter)
    2. A list of event dicts within the segment (queried from DuckDB by event_ids)
    3. A list of context event dicts BEFORE the segment (same session, preceding events)

    And produces a dict matching orchestrator-episode.schema.json.

    **Observation derivation (from context_events):**
    - `repo_state.changed_files`: Accumulate files_touched from recent tool_use/tool_result context events
    - `repo_state.diff_stat`: files = len(changed_files), insertions/deletions = 0 (known data gap)
    - `quality_state.tests.status`: Most recent T_TEST event status (pass/fail/not_run/unknown)
    - `quality_state.lint.status`: Most recent T_LINT event status
    - `context.recent_summary`: Concatenate last 3 text snippets from context events
    - `context.open_questions`: Empty list (no reliable extraction in Phase 2)
    - `context.constraints_in_force`: Empty list (populated in Phase 3)

    **Action derivation (from start trigger event):**
    - `mode`: Infer from start event text using config.mode_inference keywords with priority ordering
    - `goal`: Start event text (truncated to 500 chars)
    - `scope.paths`: Extract file paths mentioned in start event text (regex: paths with / or file extensions)
    - `executor_instruction`: Full start event text
    - `gates`: Extract from start event text using config.gate_patterns
    - `risk`: Compute from mode + scope (Implement/Integrate with protected paths = high; Explore/Plan = low)

    **Outcome derivation (from body events within segment):**
    - `executor_effects.tool_calls_count`: Count tool_use + tool_result events
    - `executor_effects.files_touched`: Accumulate from body events
    - `executor_effects.commands_ran`: From tool_use events (capped at 20)
    - `executor_effects.git_events`: From T_GIT_COMMIT events with commit hash
    - `quality.tests_status/lint_status`: From T_TEST/T_LINT body events
    - `quality.diff_stat`: files = len(files_touched), insertions/deletions = 0
    - `reward_signals.objective`: tests (1.0/0.0/0.5), lint (1.0/0.0/0.5), diff_risk (min(1.0, files * 0.1))

    **Provenance (from all events):**
    - sources: Deduplicated list of {type, ref} from event source_system/source_ref
    - Also include git commit refs from event links

    **Episode ID:** Deterministic hash: SHA-256(session_id + segment_id + config_hash)[:16]

    **Test cases:**
    - Empty context events -> observation has unknown statuses, empty changed_files, "Session start" summary
    - Context with T_TEST pass event -> observation.quality_state.tests.status = "pass"
    - Context with files_touched -> observation.repo_state.changed_files populated
    - Start event with "implement" keyword -> mode = "Implement"
    - Start event with "debug this" keyword -> mode = "Triage"
    - Start event with file paths in text -> scope.paths extracted
    - Body events with tool_use -> outcome.executor_effects.tool_calls_count > 0
    - Body events with T_GIT_COMMIT -> outcome.executor_effects.git_events populated
    - Segment with outcome "success" and T_TEST -> tests_status = "pass"
    - Populated episode passes EpisodeValidator
    - Multiple events with same source_ref -> provenance deduplicates
  </behavior>
  <implementation>
    Create `EpisodePopulator` class with:
    - `__init__(self, config: PipelineConfig)` -- stores config, builds mode inference lookup
    - `populate(self, segment: EpisodeSegment, events: list[dict], context_events: list[dict]) -> dict` -- main entry point
    - Private helpers:
      - `_derive_observation(context_events, start_event) -> dict`
      - `_derive_action(start_event, segment) -> dict`
      - `_derive_outcome(body_events, segment) -> dict`
      - `_build_provenance(events, segment) -> dict`
      - `_infer_mode(text, start_trigger) -> tuple[str, float]` -- returns (mode, confidence)
      - `_extract_scope_paths(text) -> list[str]` -- regex for file paths
      - `_extract_gates(text) -> list[dict]` -- match gate patterns from config
      - `_compute_risk(mode, scope_paths) -> str` -- low/medium/high/critical
      - `_parse_payload(payload) -> dict` -- handles JSON string or dict
      - `_make_episode_id(segment) -> str` -- deterministic hash
      - `_get_project_ref(events) -> dict` -- extract repo path from events

    Mode inference: Iterate config.mode_inference entries sorted by priority (ascending = higher priority). For each mode, check if any keyword matches the text (word-boundary regex, case-insensitive). First match wins. Default to "Implement" if no match.

    Risk computation:
    - "Explore", "Plan" -> "low"
    - "Verify", "Triage" -> "low" (usually)
    - "Implement" -> "medium" (default), "high" if protected paths in scope
    - "Integrate" -> "medium" (default), "high" if push/merge operations
    - "Refactor" -> "medium"
    - If any scope path matches config.risk_model.protected_paths -> bump one level

    Use observation_context_events and observation_context_seconds from config to limit lookback window.
  </implementation>
</feature>
