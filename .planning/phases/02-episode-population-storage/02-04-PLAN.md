**Add `write_episodes()` to `src/pipeline/storage/writer.py`:**

Implement episode writing using DuckDB MERGE for incremental upserts (an improvement over the staging table pattern used for events in Phase 1).

Function signature:
```python
def write_episodes(
    conn: duckdb.DuckDBPyConnection,
    episodes: list[dict],  # Episode dicts (from populator, after validation)
) -> dict[str, int]:  # {inserted, updated, total}
```

For each episode dict:
1. Extract flat queryable fields: `mode` from `orchestrator_action.mode`, `risk` from `orchestrator_action.risk`, `reaction_label` from `outcome.reaction.label` (or None), `reaction_confidence` from `outcome.reaction.confidence` (or None), `outcome_type` from segment outcome
2. Build the STRUCT value for observation column -- must match the STRUCT definition in the schema exactly
3. Serialize `orchestrator_action`, `outcome`, `provenance`, `labels` as JSON strings for JSON columns
4. Extract `source_files` from provenance sources as VARCHAR[]
5. Use `config_hash` from the episode dict or provenance

The MERGE statement pattern:
```sql
MERGE INTO episodes AS target
USING (SELECT ? as episode_id) AS source
ON target.episode_id = source.episode_id
WHEN MATCHED THEN UPDATE SET
    mode = ?, risk = ?, reaction_label = ?, reaction_confidence = ?,
    orchestrator_action = CAST(? AS JSON), outcome = CAST(? AS JSON),
    provenance = CAST(? AS JSON), updated_at = current_timestamp
WHEN NOT MATCHED THEN INSERT (...)
    VALUES (...)
```

NOTE: For the observation STRUCT column, DuckDB requires inserting STRUCT values via a subquery or struct_pack(). Use:
```sql
struct_pack(
    repo_state := struct_pack(
        changed_files := CAST(? AS VARCHAR[]),
        diff_stat := struct_pack(files := ?, insertions := ?, deletions := ?)
    ),
    quality_state := struct_pack(
        tests_status := ?,
        lint_status := ?,
        build_status := ?
    ),
    context := struct_pack(
        recent_summary := ?,
        open_questions := CAST(? AS VARCHAR[]),
        constraints_in_force := CAST(? AS VARCHAR[])
    )
)
```

If STRUCT insertion proves problematic with parameter binding, fall back to inserting the observation as a JSON column and casting, or use a staging table with flat columns + SQL struct_pack in the INSERT SELECT.

**Step 9 (new): Populate episodes**
```python
from src.pipeline.populator import EpisodePopulator
from src.pipeline.reaction_labeler import ReactionLabeler
from src.pipeline.episode_validator import EpisodeValidator
from src.pipeline.storage.writer import write_episodes, read_events
```

For each segment:
1. Query events within the segment: `read_events(conn, session_id=session_id)` then filter by event_ids in segment.events
2. Query context events: Events from same session with ts_utc < segment.start_ts (up to config.episode_population.observation_context_events)
3. Call `populator.populate(segment, segment_events, context_events)` to get episode dict
4. Find the next human message after segment.end_ts for reaction labeling
5. Call `reaction_labeler.label(next_human_msg, segment.end_trigger, segment.outcome)` to get reaction
6. If reaction is not None, add to episode dict under `outcome.reaction`
7. Add `session_id` and `segment_id` to the episode dict for storage
8. Add `outcome_type` from segment.outcome

**Step 10 (new): Validate episodes**
- For each populated episode, call `validator.validate(episode_dict)`
- Valid episodes go to write list; invalid episodes are logged with errors
- Track counts: valid, invalid, validation_errors

**Step 11 (new): Write episodes to DuckDB**
- Call `write_episodes(conn, valid_episodes)`
