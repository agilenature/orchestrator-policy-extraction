Update `src/pipeline/ebc/detector.py` to add a secondary signal type: `high_read_ratio`.

**Concept:** A session that is heavily read-dominant (Read/Glob/Grep) with very few writes may indicate Discovery Mode even if the writes that do occur are within the contract. This signal supplements the primary file-set comparison.

Add method:
```python
def _compute_tool_ratio_signal(self, session_events: list[dict]) -> DriftSignal | None:
    """Detect high read-to-write ratio indicating possible Discovery Mode.

    Returns a DriftSignal if the read/write ratio exceeds 10:1 AND
    there are at least 20 read-class events (avoid firing on tiny sessions).
    """
```

Algorithm:
1. Count read-class events: tool_name in {"Read", "Glob", "Grep", "WebFetch", "WebSearch"}.
2. Count write-class events: tool_name in self._write_tool_names or (tool_name == "Bash" and command contains write indicators).
3. If read_count < 20: return None (session too small for ratio signal).
4. If write_count == 0 and read_count >= 20: return DriftSignal(signal_type="high_read_ratio", detail=f"read={read_count}, write=0 — session appears exploratory", weight=0.5).
5. If write_count > 0 and read_count / write_count > 10: return DriftSignal(signal_type="high_read_ratio", detail=f"read/write ratio {read_count/write_count:.1f}:1 — heavily read-dominant", weight=0.3).
6. Otherwise return None.

Integrate into `detect()` method:
After computing file-set signals, call `_compute_tool_ratio_signal(session_events)`. If it returns a signal, append it to the signals list. This contributes to drift_score alongside file-set signals.

**Important:** The tool ratio signal alone should NOT trigger a drift alert if there are no file-set signals. Adjust the score calculation: if ALL signals are `high_read_ratio` type (no file-set signals), require a higher effective threshold using `config.ebc_drift.ratio_only_threshold` (default 0.8). Implementation: add a flag `has_file_signals = any(s.signal_type != "high_read_ratio" for s in signals)`. If not has_file_signals and drift_score < self._ratio_only_threshold: return None.

**Config prerequisite (already in Plan 01 EBCDriftConfig):** This task reads `self._ratio_only_threshold` which is set from `config.ebc_drift.ratio_only_threshold` in `__init__`.

Add tests to `tests/test_ebc_integration.py` (or a separate section at the end):
- Test high read ratio with 30 reads, 0 writes: returns signal with weight 0.5
- Test high read ratio with 30 reads, 2 writes (ratio 15:1): returns signal with weight 0.3
- Test read ratio below 10:1 (20 reads, 5 writes): returns None
- Test small session (5 reads, 0 writes): returns None (below minimum threshold)
- Test ratio signal alone does not trigger alert at default threshold 0.5 (only fires at config.ebc_drift.ratio_only_threshold=0.8)
- Test ratio signal combined with file-set signals increases drift_score
