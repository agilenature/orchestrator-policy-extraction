**1. Implement `src/pipeline/rag/retriever.py`:**

```python
class HybridRetriever:
    def __init__(self, conn: duckdb.DuckDBPyConnection, top_k: int = 5):
        self._conn = conn
        self._top_k = top_k

    def retrieve(self, query_text: str, query_embedding: list[float],
                 exclude_episode_id: str | None = None) -> list[dict]:
        bm25_results = self._bm25_search(query_text, exclude_episode_id)
        embedding_results = self._embedding_search(query_embedding, exclude_episode_id)
        return self._rrf_fuse(bm25_results, embedding_results, k=self._top_k)
```

Follow the exact patterns from 05-RESEARCH.md Pattern 1:
- `_bm25_search`: Use `fts_main_episode_search_text.match_bm25(episode_id, ?)` with parameterized query. Use exclude_clause if exclude_episode_id provided. Fetch top_k * 2 results (over-fetch for fusion). Use parameterized queries (not f-string interpolation for the exclude clause -- use WHERE clause with CASE or conditional).
- `_embedding_search`: Use `array_cosine_similarity(embedding, ?::FLOAT[384])`. Same exclude pattern. Fetch top_k * 2 results.
- `_rrf_fuse`: Static method. RRF score = sum(1/(rrf_k + rank)) with rrf_k=60. Return list of dicts with episode_id and rrf_score, sorted by score descending, limited to k.

**Important:** For the BM25 exclude clause, use SQL parameter binding:
```sql
WHERE score IS NOT NULL AND (? IS NULL OR episode_id != ?)
```
passing `[query, exclude_id, exclude_id]`. This avoids SQL injection.
