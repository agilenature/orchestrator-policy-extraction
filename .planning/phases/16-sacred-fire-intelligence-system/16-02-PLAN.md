**1. compute_te_for_session(conn, session_id) -> list[dict]:**
Computes TE for all subjects present in a session. Returns list of row dicts ready for INSERT.

SQL query (per Q1 locked decision, parameterized by subject):
```sql
SELECT
    session_id,
    subject,
    human_id,
    MAX(marker_level) / 7.0 AS raven_depth,
    CAST(COUNT(*) FILTER (WHERE axis_identified IS NOT NULL) AS FLOAT)
        / NULLIF(COUNT(*), 0) AS crow_efficiency,
    CAST(COUNT(*) FILTER (WHERE flood_confirmed = true) AS FLOAT)
        / NULLIF(COUNT(*), 0) AS transport_speed,
    0.5 AS trunk_quality,  -- pending sentinel per Q2
    'pending' AS trunk_quality_status
FROM flame_events
WHERE session_id = ?
GROUP BY session_id, subject, human_id
```

For each row:
- Compute composite_te = raven_depth * crow_efficiency * transport_speed * trunk_quality (all 0-1, no inversion per Q1)
- Generate te_id = SHA-256(session_id + subject)[:16]
- Include fringe_drift_rate from compute_fringe_drift()

Return list of dicts with all transport_efficiency_sessions columns.

**2. compute_fringe_drift(conn, session_id, subject) -> float | None:**
Per Q3 locked decision:
```sql
SELECT
    CASE WHEN COUNT(*) FILTER (WHERE marker_level IN (1, 2)) > 0
        THEN CASE WHEN COUNT(*) FILTER (WHERE marker_level >= 6 AND flood_confirmed = true) > 0
            THEN 0.0
            ELSE 1.0
        END
        ELSE NULL
    END AS fringe_drift_rate
FROM flame_events
WHERE session_id = ? AND subject = ?
```
Returns float or None.

**3. write_te_rows(conn, rows: list[dict]) -> int:**
INSERT OR REPLACE into transport_efficiency_sessions. Returns count of rows written.

**4. backfill_trunk_quality(conn) -> int:**
Per Q2 locked decision. Runs during each pipeline execution:
- Find pending rows: `SELECT te_id, session_id, subject, human_id FROM transport_efficiency_sessions WHERE trunk_quality_status = 'pending'`
- For each pending row, count sessions with >= 3 newer sessions for same (human_id, subject):
  ```sql
  SELECT COUNT(DISTINCT tes2.session_id)
  FROM transport_efficiency_sessions tes2
  WHERE tes2.human_id = ? AND tes2.subject = ?
    AND tes2.created_at > ?
  ```
- If count >= 3: compute actual trunk_quality:
  ```sql
  -- For each Level 0 flame_event with axis_identified in original session:
  -- Count distinct sessions in next 3 where that axis reappears at Level >= 5
  SELECT
    CAST(COUNT(DISTINCT fe2.session_id) AS FLOAT) / 3.0
  FROM flame_events fe_orig
  JOIN flame_events fe2 ON fe2.axis_identified = fe_orig.axis_identified
    AND fe2.marker_level >= 5
    AND fe2.session_id != fe_orig.session_id
    AND fe2.subject = fe_orig.subject
  WHERE fe_orig.session_id = ? AND fe_orig.subject = ? AND fe_orig.marker_level = 0
    AND fe2.session_id IN (
      SELECT DISTINCT session_id FROM transport_efficiency_sessions
      WHERE human_id = ? AND subject = ? AND created_at > ?
      ORDER BY created_at LIMIT 3
    )
  ```
  - If no Level 0 events: trunk_quality stays 0.5
  - UPDATE trunk_quality, trunk_quality_status='confirmed', recalculate composite_te
- Returns count of rows backfilled.

**5. backfill_te_delta(conn) -> int:**
Per Q6 locked decision:
- Find accepted memory_candidates with NULL te_delta: `SELECT id, created_at FROM memory_candidates WHERE status='validated' AND te_delta IS NULL`
- For each: compute pre_te_avg = AVG(composite_te) for AI subject, 5 sessions BEFORE created_at
- Check if >= 5 post-acceptance AI sessions exist. If so:
  - post_te_avg = AVG(composite_te) for AI subject, 5 sessions AFTER reviewed_at
  - te_delta = post_te_avg - pre_te_avg
  - UPDATE memory_candidates SET pre_te_avg=?, post_te_avg=?, te_delta=? WHERE id=?
- Returns count of candidates backfilled.
