**src/pipeline/wisdom/ingestor.py — WisdomIngestor class:**

```python
class WisdomIngestor:
    def __init__(self, store: WisdomStore, embedder: EpisodeEmbedder):
        self._store = store
        self._embedder = embedder
```

Methods:

1. `ingest_from_file(json_path: str, conn: duckdb.DuckDBPyConnection) -> dict`:
   - Open and parse JSON file (json.load). File is a JSON array of entry objects.
   - For each entry:
     a. Generate wisdom_id via `generate_wisdom_id(entry["type"], entry["title"], entry.get("source_document", ""))`
     b. Generate content_for_embedding via `generate_content_for_embedding(entry["type"], entry["title"], entry["description"])`
     c. Build WisdomEntity from entry fields + generated wisdom_id + content_for_embedding
     d. Generate embedding via `self._embedder.embed_text(content_for_embedding)`
     e. Call `self._store.insert(entity, embedding)`
   - After all entries processed, call `self._store.rebuild_indexes()`
   - Return stats dict: `{"ingested": count, "total_in_file": len(entries)}`
   - Handle errors gracefully: if a single entry fails (e.g., missing required field), log warning and continue. Track `skipped` count in return stats.

2. Static helper `validate_entry(entry: dict) -> list[str]`:
   - Returns list of error strings for invalid entries
   - Checks: type is one of 4 valid values, title is non-empty string, description is non-empty string
   - Used internally before processing each entry

**seed_wisdom.json format** (per locked decision, JSON not YAML):
```json
[
  {
    "type": "breakthrough|dead_end|scope_decision|method_decision",
    "title": "Short descriptive title",
    "description": "Full description of the wisdom",
    "confidence_score": 0.9,
    "source_document": "docs/analysis/objectivism-knowledge-extraction/<filename>.md",
    "episode_ids": null,
    "metadata": {}
  }
]
```

Note: wisdom_id is NOT in the JSON file -- it is computed during ingestion from type|title|source_document. content_for_embedding is also computed during ingestion.
