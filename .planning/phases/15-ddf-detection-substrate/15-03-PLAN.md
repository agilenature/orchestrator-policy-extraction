**flame_extractor.py** -- `FlameEventExtractor`:

The Tier 2 extractor enriches Tier 1 stub detections and performs L3-7 scoring. For Phase 15, this uses heuristic scoring (NOT LLM calls). The LLM integration point is designed but stubbed -- the method signature accepts episode context and returns scored FlameEvents. Future phases can swap in LLM scoring.

Class `FlameEventExtractor`:
- Constructor: `__init__(self, config: PipelineConfig, conn: duckdb.DuckDBPyConnection)`
- `enrich_tier1(self, session_id: str, episodes: list[dict]) -> list[FlameEvent]`:
  - Read existing Tier 1 flame_events from DuckDB for this session
  - For each Tier 1 event, use episode context to upgrade marker_level:
    - L0 stub + episode has multiple scope paths -> L3 (cross-context application)
    - L1 stub + episode has constraint with 3+ sessions honored -> L4 (principle identification)
    - L2 stub + episode has escalation or correction -> L5 (deep principle naming)
    - Any stub + episode reaction is 'approve' + marker has axis_identified + flood_confirmed = true -> L6 (Concretization Flood)
  - Return upgraded FlameEvent records with detection_source='opeml'

- `detect_ai_markers(self, session_id: str, episodes: list[dict], tagged_events: list) -> list[FlameEvent]`:
  - Scan AI (executor) assistant_text events for DDF patterns:
    - AI introduces a new CCD axis not in session context -> subject='ai', marker_level=2
    - AI performs causal isolation (references foil or counterfactual reasoning) -> subject='ai', marker_level=3
    - AI generates a Concretization Flood (3+ concrete examples of one principle in one message) -> subject='ai', marker_level=6
  - Uses same regex patterns as Tier 1 L0-2 but applied to executor events with subject='ai'
  - Returns FlameEvent list with subject='ai', detection_source='opeml'

- `deposit_level6(self, conn: duckdb.DuckDBPyConnection, events: list[FlameEvent]) -> int`:
  - Filter events where marker_level >= 6 AND flood_confirmed = True
  - For each, call deposit_to_memory_candidates:
    - ccd_axis: event.axis_identified (required for Level 6)
    - scope_rule: derive from episode scope_paths or "Applies to sessions where this axis is active"
    - flood_example: event.evidence_excerpt
    - source_flame_event_id: event.flame_event_id
    - pipeline_component: "ddf_tier2"
    - fidelity: 2
  - Call mark_deposited for each successfully deposited event
  - Return count of deposits made

**causal_isolation.py** -- `CausalIsolationRecorder`:
Per CONTEXT.md Q7: passive recorder that reads FoilInstantiator results.

Class `CausalIsolationRecorder`:
- Constructor: `__init__(self, conn: duckdb.DuckDBPyConnection)`
- `record(self, session_id: str) -> list[FlameEvent]`:
  - Query premise_registry for this session's premises that have foil_path_outcomes (non-null, non-empty JSON)
  - For each premise with foil results:
    - Parse foil_path_outcomes JSON
    - If outcomes indicate successful isolation (divergence_node present with evidence) -> FlameEvent(marker_level=3, marker_type='causal_isolation_success', flood_confirmed=True)
    - If outcomes indicate failed foil (no divergence, inconclusive) -> FlameEvent(marker_level=2, marker_type='causal_isolation_failed', flood_confirmed=False)
  - For premises WITHOUT foil_path_outcomes where the claim contains causal language (reuse L1/L2 patterns):
    - FlameEvent(marker_level=1, marker_type='missing_isolation', flood_confirmed=False)
    - This flags potential Post Hoc Ergo Propter Hoc (temporal-only causal claims)
  - Return all FlameEvent records with subject='human', detection_source='opeml'

**false_integration.py** -- `FalseIntegrationDetector`:
Per CONTEXT.md Q4: LLM hypothesize-and-check stubbed as heuristic for Phase 15.

Class `FalseIntegrationDetector`:
- Constructor: `__init__(self, config: PipelineConfig, conn: duckdb.DuckDBPyConnection)`
- `detect(self, session_id: str, episodes: list[dict]) -> tuple[list[FlameEvent], list[AxisHypothesis]]`:
  - For each episode with constraints or escalation data:
    - Extract scope_paths and constraint texts
    - Heuristic: if episode touches 2+ distinct scope path prefixes AND constraint text contains conflicting keywords -> hypothesize false integration
    - Create AxisHypothesis with hypothesized_axis from constraint text, confidence from scope overlap metric
    - If confidence >= config.ddf.false_integration_confidence_threshold (0.6):
      - Emit FlameEvent(marker_type='false_integration', marker_level=5, subject='ai', detection_source='opeml') in ai_flame_events
    - If confidence < 0.6:
      - Log as 'false_integration_inconclusive' (no flame event emitted)
  - Write AxisHypothesis records to axis_hypotheses table
  - Return (flame_events, hypotheses)

- `_write_hypotheses(self, hypotheses: list[AxisHypothesis]) -> int`:
  - INSERT OR REPLACE into axis_hypotheses table
  - Return count
