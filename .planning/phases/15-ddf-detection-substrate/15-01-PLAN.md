**schema.py** -- `create_ddf_schema(conn)` function containing:

1. `flame_events` table (single table for both human and AI, per RESEARCH recommendation):
   - flame_event_id VARCHAR PRIMARY KEY
   - session_id VARCHAR NOT NULL
   - human_id VARCHAR (nullable -- default_human heuristic for single-user)
   - prompt_number INTEGER (nullable)
   - marker_level INTEGER NOT NULL (0-7)
   - marker_type VARCHAR NOT NULL
   - evidence_excerpt TEXT (nullable)
   - quality_score FLOAT (nullable)
   - axis_identified VARCHAR (nullable)
   - flood_confirmed BOOLEAN DEFAULT FALSE
   - subject VARCHAR NOT NULL DEFAULT 'human' CHECK (subject IN ('human', 'ai'))
   - detection_source VARCHAR NOT NULL DEFAULT 'stub' CHECK (detection_source IN ('stub', 'opeml'))
   - deposited_to_candidates BOOLEAN DEFAULT FALSE
   - source_episode_id VARCHAR (nullable)
   - session_event_ref VARCHAR (nullable)
   - created_at TIMESTAMPTZ DEFAULT NOW()
   - Indexes: idx_flame_session, idx_flame_level, idx_flame_subject, idx_flame_marker_type

2. `ai_flame_events` as a VIEW: `CREATE VIEW IF NOT EXISTS ai_flame_events AS SELECT * FROM flame_events WHERE subject = 'ai'`
   NOTE: DuckDB may not support `CREATE VIEW IF NOT EXISTS`. Use `CREATE OR REPLACE VIEW` instead.

3. `axis_hypotheses` table (for Wave 2 DDF-09 False Integration):
   - hypothesis_id VARCHAR PRIMARY KEY
   - session_id VARCHAR NOT NULL
   - episode_id VARCHAR
   - hypothesized_axis VARCHAR NOT NULL
   - confidence FLOAT NOT NULL
   - marker_type VARCHAR NOT NULL DEFAULT 'false_integration'
   - evidence TEXT
   - created_at TIMESTAMPTZ DEFAULT NOW()

4. `constraint_metrics` table (for Wave 3 DDF-05 GeneralizationRadius):
   - constraint_id VARCHAR PRIMARY KEY
   - radius INTEGER NOT NULL DEFAULT 0
   - firing_count INTEGER NOT NULL DEFAULT 0
   - is_stagnant BOOLEAN DEFAULT FALSE
   - last_computed TIMESTAMPTZ DEFAULT NOW()

5. Extend `memory_candidates` with 3 new columns via ALTER TABLE + try/except:
   - source_flame_event_id VARCHAR (nullable)
   - fidelity INTEGER DEFAULT 2 (use DEFAULT only, no NOT NULL -- per Pitfall 5)
   - detection_count INTEGER DEFAULT 1 (use DEFAULT only, no NOT NULL)
