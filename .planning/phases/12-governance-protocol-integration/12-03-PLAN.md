  <action>
Create `src/pipeline/governance/stability.py` with the stability check runner.

1. **StabilityOutcome dataclass:**
   ```python
   @dataclass
   class StabilityOutcome:
       run_id: str
       check_id: str
       status: str  # "pass", "fail", "error"
       exit_code: int
       stdout: str
       stderr: str
       started_at: str  # ISO 8601
       ended_at: str    # ISO 8601
       session_id: str | None = None
       actor_name: str | None = None
       actor_email: str | None = None
   ```

2. **StabilityRunner class:**
   - `__init__(conn: duckdb.DuckDBPyConnection, config: GovernanceConfig)`:
     Store connection and config. Cache actor info by calling `_get_git_actor()` once.
   - `_get_git_actor() -> tuple[str | None, str | None]`:
     Run `subprocess.run(["git", "config", "user.name"], capture_output=True, text=True, timeout=5)` and `subprocess.run(["git", "config", "user.email"], ...)`. Return (name, email) or (None, None) on error. Cache result.
   - `run_checks(repo_root: str | Path, session_id: str | None = None) -> list[StabilityOutcome]`:
     For each check in `config.stability_checks`:
     1. Generate run_id = `str(uuid.uuid4())`
     2. Record started_at = `datetime.now(timezone.utc).isoformat()`
     3. Execute via `subprocess.run(check.command, capture_output=True, text=True, timeout=check.timeout_seconds, cwd=str(repo_root))`
     4. Record ended_at
     5. Determine status: returncode == 0 -> "pass", else -> "fail"
     6. Build StabilityOutcome
     7. Catch `subprocess.TimeoutExpired` EXPLICITLY (per pitfall 4): status="error", exit_code=-1, stderr=f"Timeout after {check.timeout_seconds}s"
     8. Catch generic Exception: status="error", exit_code=-1, stderr=str(e)
     9. Truncate stdout/stderr to 10000 chars each to avoid DuckDB storage bloat
     After all checks:
     10. Write all outcomes to DuckDB: `INSERT INTO stability_outcomes VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)`
     11. Return list of outcomes
   - `flag_missing_validation(conn: duckdb.DuckDBPyConnection) -> int`:
     Static method or instance method. Query episodes:
     ```sql
     UPDATE episodes
     SET stability_check_status = 'missing'
     WHERE requires_stability_check = TRUE
       AND stability_check_status IS NULL
     ```
     Return count of rows updated.
   - `mark_validated(conn: duckdb.DuckDBPyConnection) -> int`:
     After checks pass, mark flagged episodes as validated:
     ```sql
     UPDATE episodes
     SET stability_check_status = 'validated'
     WHERE requires_stability_check = TRUE
       AND (stability_check_status IS NULL OR stability_check_status = 'missing')
     ```
     Return count of rows updated.

   Import `subprocess`, `uuid`, `duckdb`, `datetime`, `pathlib.Path` from stdlib. Import `GovernanceConfig`, `StabilityCheckDef` from `src.pipeline.models.config`.
  </action>

  </verify>

    StabilityRunner class with run_checks(), flag_missing_validation(), mark_validated(). StabilityOutcome dataclass. Subprocess execution with explicit timeout handling.
