## Decisions Made

- FrontierChecker uses `sorted(set(active_axes))` + `combinations` for deterministic pair ordering matching the EdgeGenerator pattern from 16.1-02
- CrossAxisVerifier foil level threshold uses strict `>` (edge_level > premise_level + 1), not `>=`, so an edge one level above the premise is acceptable
- Activation matching with `goal_type=["any"]` matches any current goal_type including None -- consistent with EdgeWriter convention
- PAG gate axis extraction uses `memory_candidates.ccd_axis` for known axis lookup with case-insensitive substring matching against premise claim and scope text
- New checks (5 and 6) added after existing checks (1-4) in main() to maintain backward compatibility
