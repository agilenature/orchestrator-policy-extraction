Capture tool provenance events from the OpenClaw Gateway WebSocket connection during task execution and store them as episode_events in SQLite.

Purpose: Implements MC-02 -- tool provenance recording. During task execution, the agent makes tool calls (Read, Edit, Bash, Write, Grep, Glob), touches files, runs commands, and produces test/git results.   <name>Task 1: ProvenanceCapture adapter for Gateway WebSocket</name>
  <files>
    mission-control/src/lib/openclaw/provenance.ts
  </files>
  <action>
    Create the ProvenanceCapture class that hooks into Mission Control's existing OpenClaw Gateway WebSocket client to capture tool provenance events.

    **IMPORTANT: The Gateway WebSocket API is NOT fully documented (research confidence: LOW).** Build this as a thin, defensive adapter that logs unrecognized messages instead of failing.

    **provenance.ts:**
    - Define ToolProvenanceEvent interface: { event_id: string, episode_id: string, timestamp: string, received_at: string, event_type: 'tool_call' | 'tool_result' | 'file_touch' | 'command_run' | 'test_result' | 'git_event' | 'lint_result' | 'build_result', payload: Record<string, unknown> }
    - Define GatewayMessage interface broadly: { type?: string, content?: any, tool_name?: string, tool_input?: Record<string, unknown>, tool_result?: string, [key: string]: unknown } -- keep it flexible since the protocol is not fully documented
    - Export class ProvenanceCapture:
      - Constructor: takes db (better-sqlite3 Database) and optional options { batchIntervalMs?: number (default 100), maxBatchSize?: number (default 50) }
      - Property: currentEpisodeId: string | null -- set when task execution starts, cleared when complete
      - Method `startCapture(episodeId: string): void` -- sets currentEpisodeId, clears event buffer
      - Method `stopCapture(): void` -- flushes remaining buffer, clears currentEpisodeId
      - Method `onGatewayMessage(message: GatewayMessage): void` -- the main hook, called for each WS message:
        1. If currentEpisodeId is null, return (not capturing)
        2. Parse message to identify tool-related events:
           - If message has content array with blocks of type 'tool_use' -> create 'tool_call' event for each block with payload: { tool_name: block.name, tool_input: block.input }
           - If message type is 'tool_result' or has tool_result field -> create 'tool_result' event with payload: { tool_name, result: truncated to 1000 chars }
           - Derive event_type from tool_name: Bash/command -> 'command_run', Read/Edit/Write/Glob/Grep -> 'file_touch', git commands -> 'git_event', pytest/test -> 'test_result'
        3. For unrecognized message types: log with console.debug (not warn) to avoid noise, store as 'unknown' event type if message has content
        4. Add event to buffer
        5. If buffer >= maxBatchSize, flush
      - Method `flush(): void` -- write all buffered events to SQLite in a single transaction (INSERT batch). Use db.transaction() for atomicity. This prevents the SQLite write contention described in research Pitfall 3.
      - Private `classifyTool(toolName: string): string` -- maps tool names to event_types: { Bash: 'command_run', Read: 'file_touch', Edit: 'file_touch', Write: 'file_touch', Glob: 'file_touch', Grep: 'file_touch' }. Default to 'tool_call' for unrecognized tools.
    - Use Gateway timestamp if available in the message, otherwise fall back to new Date().toISOString() for `timestamp`. Always record `received_at` as new Date().toISOString(). This follows research guidance on Pitfall 5 (timestamp ordering).
    - Set up a periodic flush timer (setInterval at batchIntervalMs) in startCapture(), clear in stopCapture()
  </action>
  <verify>
    File compiles without TypeScript errors. ProvenanceCapture class has startCapture, stopCapture, onGatewayMessage, flush methods. GatewayMessage interface is flexible. Batch buffering and transactional writes are implemented.
  </verify>
  <done>
    ProvenanceCapture adapter hooks into Gateway WebSocket messages, classifies tool events, buffers them, and writes to episode_events in batched transactions. Handles unrecognized messages defensively. Uses Gateway timestamps for canonical ordering.
  </done>

<task type="auto">
  <name>Task 2: ProvenanceAggregator for episode outcome population</name>
  <files>
    mission-control/src/lib/episodes/provenance-aggregator.ts
  </files>
  <action>
    Create the ProvenanceAggregator that summarizes raw provenance events into the episode's outcome.executor_effects structure.

    **provenance-aggregator.ts:**
    - Import getEpisodeEvents from '../db/episodes'
    - Define ExecutorEffects interface (matching Pydantic ExecutorEffects model): { tool_calls_count: number, files_touched: string[], commands_ran: string[], git_events: Array<{ type: string, ref?: string, message?: string }> }
    - Define OutcomeQuality interface: { tests_status: string, lint_status: string, diff_stat: { files: number, insertions: number, deletions: number } }
    - Export class ProvenanceAggregator:
      - Constructor: takes db (better-sqlite3 Database)
      - Method `aggregate(episodeId: string): { executorEffects: ExecutorEffects, quality: OutcomeQuality }`:
        1. Fetch all episode_events for the episode via getEpisodeEvents()
        2. Count tool_call events -> tool_calls_count
        3. Collect unique file paths from file_touch events (extract from payload.tool_input.file_path or payload.files_touched) -> files_touched
        4. Collect commands from command_run events (extract from payload.tool_input.command, truncate to 200 chars) -> commands_ran
        5. Collect git events (type, ref, message from git_event payloads) -> git_events
        6. Extract latest test_result event -> tests_status ('pass'/'fail'/'error'/'unknown')
        7. Extract latest lint_result event -> lint_status ('pass'/'fail'/'unknown')
        8. Compute diff_stat from git_events if available, otherwise { files: files_touched.length, insertions: 0, deletions: 0 }
        9. Return aggregated structures
      - Method `updateEpisodeOutcome(episodeId: string): void`:
        1. Call aggregate(episodeId)
        2. Read current episode outcome JSON
        3. Merge executor_effects and quality into the outcome
        4. UPDATE episodes SET outcome = JSON.stringify(merged), updated_at = datetime('now')
    - All methods should be null-safe: if no events exist, return zeroed/empty structures
  </action>
  <verify>
    File compiles without TypeScript errors. ProvenanceAggregator has aggregate() and updateEpisodeOutcome() methods. ExecutorEffects interface matches Pydantic model field names (snake_case).
  </verify>
  <done>
    ProvenanceAggregator reads raw episode_events from SQLite, computes aggregate executor_effects (tool counts, files touched, commands, git events) and quality metrics (test/lint status, diff stats), and writes the summary back to the episode's outcome JSON column.
  </done>
