```python
class ScenarioSpec(BaseModel, frozen=True):
    """Specification for an assessment scenario.

    Generated from project_wisdom entries. Contains the scenario
    briefing and broken implementation file content.
    """
    scenario_id: str  # SHA-256[:16] of wisdom_id + ddf_target_level
    wisdom_id: str  # Source project_wisdom entry
    ddf_target_level: int  # 1-7
    entity_type: str  # dead_end, breakthrough, scope_decision
    title: str
    scenario_context: str  # Content for scenario_context.md
    broken_impl_filename: str  # e.g. "broken_impl.py"
    broken_impl_content: str  # Content for broken implementation file
    handicap_claude_md: Optional[str] = None  # L5-L7: plausible-but-wrong CLAUDE.md text
    scenario_seed: Optional[str] = None  # From project_wisdom.scenario_seed

    @field_validator("ddf_target_level")
    @classmethod
    def ddf_level_valid(cls, v: int) -> int:
        if not 1 <= v <= 7:
            raise ValueError(f"ddf_target_level must be 1-7, got {v}")
        return v

    @classmethod
    def make_id(cls, wisdom_id: str, ddf_target_level: int) -> str:
        key = f"{wisdom_id}:{ddf_target_level}"
        return hashlib.sha256(key.encode()).hexdigest()[:16]
```

```python
class AssessmentSession(BaseModel, frozen=True):
    """Tracks an assessment session lifecycle."""
    session_id: str  # Claude Code session UUID
    scenario_id: str  # Links to ScenarioSpec
    candidate_id: str  # Identifier for the candidate
    assessment_dir: str  # /tmp/ope_assess_{session_id}/
    jsonl_path: Optional[str] = None  # Derived after session creation
    status: Literal["setup", "running", "completed", "failed"] = "setup"
    handicap_level: Optional[int] = None  # DDF level the AI is handicapped to
    session_artifact_path: Optional[str] = None  # tar.gz path after cleanup
    started_at: datetime = Field(default_factory=lambda: datetime.now(timezone.utc))
    completed_at: Optional[datetime] = None

    @classmethod
    def derive_jsonl_path(cls, assessment_dir: str, session_id: str) -> str:
        """Derive JSONL path from assessment dir.

        Claude Code writes JSONL to:
        ~/.claude/projects/{cwd-with-slashes-replaced-by-dashes}/{session-uuid}.jsonl

        For /tmp/ope_assess_abc123/:
        ~/.claude/projects/-tmp-ope_assess_abc123/{session-uuid}.jsonl
        """
        import os
        # Strip trailing slash, replace / with -
        clean_dir = assessment_dir.rstrip("/")
        encoded = clean_dir.replace("/", "-")
        return os.path.expanduser(
            f"~/.claude/projects/{encoded}/{session_id}.jsonl"
        )
```

<pitfalls>
1. **DuckDB ALTER TABLE + CHECK constraint:** Do NOT add CHECK constraint on source_type in ALTER TABLE -- DuckDB does not support CHECK in ALTER TABLE. Validate in Pydantic instead.
2. **project_wisdom table creation order:** The ALTER TABLE for project_wisdom columns requires the table to exist. WisdomStore creates it independently. In tests, either create WisdomStore first or use inline DDL for project_wisdom before calling create_assessment_schema.
3. **Schema chain order:** create_assessment_schema must be called AFTER create_te_schema (which is after create_topology_schema) in create_ddf_schema. Add the import and call at the very end.
4. **JSONL path encoding:** The path encoding replaces ALL slashes with dashes, including the leading slash. So `/tmp/foo/` becomes `-tmp-foo`. Verify this in the spike.
5. **No --project-dir flag:** Claude Code does NOT have a --project-dir flag. The CWD at launch time determines the project directory. Must cd to assessment dir before launching.
</pitfalls>
