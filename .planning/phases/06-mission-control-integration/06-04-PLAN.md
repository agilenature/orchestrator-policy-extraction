    **episodes/stream/route.ts:**
    - Export GET handler that returns a ReadableStream SSE response
    - Use TransformStream pattern for Server-Sent Events:
      ```
      const stream = new TransformStream();
      const writer = stream.writable.getWriter();
      const encoder = new TextEncoder();
      ```
    - Return new Response(stream.readable, { headers: { 'Content-Type': 'text/event-stream', 'Cache-Control': 'no-cache', 'Connection': 'keep-alive' } })
    - Implement a simple event bus pattern: maintain a Set of active writers. When an episode event occurs (via a global emitter or in-memory pub/sub), broadcast to all connected writers.
    - Export a `broadcastEpisodeEvent(event: EpisodeSSEEvent)` function that other modules can call to push events to connected clients
    - EpisodeSSEEvent types: episode_created, episode_provenance, episode_reviewed, constraint_extracted (matching research Pattern 5)
    - Send keep-alive comment every 30 seconds (`:keep-alive\n\n`) matching MC's existing SSE pattern
    - Handle client disconnect: writer.closed promise removes writer from active set

    **EpisodeTimeline.tsx:**
    - 'use client' directive
    - Define EpisodeTimelineProps: { episodeId: string }
    - Use useEffect to connect to /api/episodes/stream via EventSource
    - Listen for episode_provenance events matching the episodeId prop
    - Maintain a state array of timeline entries: { timestamp, event_type, summary }
    - Render entries as a vertical timeline (Tailwind: relative border-l-2 pl-4 space-y-2)
    - Each entry: small circle on the timeline, event_type label, summary text, relative timestamp
    - Auto-scroll to latest entry
    - Also fetch existing events on mount via GET /api/episodes/{episodeId}/events for backfill
    - Cleanup: close EventSource on unmount

    - Handle duplicate constraint_id gracefully (ON CONFLICT behavior from the CRUD layer enriches examples array)
