<feature>
  <name>Trigger-Based Episode Segmenter (EXTRACT-03)</name>
  <files>src/pipeline/segmenter.py, tests/test_segmenter.py</files>
  <behavior>
    The segmenter is a state machine that walks the tagged event stream, opening episodes on start triggers and closing them on end triggers.

    **Start triggers:** O_DIR, O_GATE (per EXTRACT-03 spec)

    **End triggers** (fail-fast per Q2 locked decision):
    - T_TEST (ANY test execution, regardless of pass/fail)
    - T_RISKY (any risky command)
    - T_GIT_COMMIT (commit is a natural decision boundary)
    - X_PROPOSE (executor proposes next steps -> orchestrator must decide)
    - X_ASK (executor asks question -> orchestrator must answer)
    - O_DIR, O_GATE, O_CORR (next orchestrator action supersedes current episode)
    - 30-second timeout (Q4 locked decision) between consecutive events

    **NOT end triggers:**
    - T_LINT (observation only per Q2 locked decision, unless blocking -- but we don't detect blocking lint in Phase 1)
    - assistant_text, assistant_thinking (intermediate executor work)
    - tool_use, tool_result without T_* classification (routine tool calls)

    **Outcome determination:**
    - T_TEST with test passing in payload -> outcome='success'
    - T_TEST with test failing in payload -> outcome='failure'
    - T_TEST with unknown result -> outcome='test_executed'
    - T_RISKY -> outcome='risky_action'
    - T_GIT_COMMIT -> outcome='committed'
    - X_PROPOSE, X_ASK -> outcome='executor_handoff'
    - O_DIR, O_GATE, O_CORR (superseded) -> outcome='superseded'
    - Timeout -> outcome='timeout'
    - Stream end -> outcome='stream_end'

    **State machine logic:**
    1. Walk events in order
    2. If no episode open and event is a start trigger -> open new episode
    3. If episode open and timeout exceeded -> close with 'timeout', then check if current event is start trigger
    4. If episode open and event is a start trigger -> close current with 'superseded', open new episode with this event
    5. If episode open -> add event to current episode
    6. If episode open and event is an end trigger (and not a start trigger) -> close episode, set outcome
    7. At stream end, close any open episode with 'stream_end'

    **Edge case: Events between episodes (no open episode, not a start trigger):**
    - These events are "orphaned" -- they don't belong to any episode
    - Track orphan count for diagnostics but don't create episodes for them

    **Flat episodes with metadata (Q3 locked decision):**
    - If a user message (actor=human_orchestrator) appears mid-episode that is NOT O_DIR/O_GATE/O_CORR, increment interruption_count
    - If actor changes from executor->human->executor within episode, increment context_switches
    - If interruption_count > 0 OR context_switches > 0, set complexity='complex'

    Test cases (input event sequences -> expected segments):

    Basic segmentation:
    - [O_DIR, tool_use, tool_result(test), T_TEST] -> 1 episode, start=O_DIR, end=T_TEST
    - [O_GATE, assistant_text, X_PROPOSE] -> 1 episode, start=O_GATE, end=X_PROPOSE
    - [O_DIR, tool_use, T_LINT, tool_use, T_TEST] -> 1 episode (T_LINT does NOT end it)
    - [O_DIR, ...30s gap..., tool_use] -> 1 episode ended by timeout

    Superseding:
    - [O_DIR, assistant_text, O_GATE, assistant_text] -> 2 episodes (O_GATE supersedes first)

    Fail-fast:
    - [O_DIR, assistant_text, T_TEST(pass)] -> 1 episode, outcome=success
    - [O_DIR, assistant_text, T_TEST(fail)] -> 1 episode, outcome=failure
    - [O_DIR, tool_use, T_RISKY] -> 1 episode, outcome=risky_action

    Timeout:
    - Events with >30s gap -> episode closes with timeout outcome
    - Timeout value from config.episode_timeout_seconds (NOT hardcoded)

    Complexity metadata:
    - [O_DIR, executor_text, user_msg(non-correction), executor_text, T_TEST] -> complexity=complex, interruption_count=1
  </behavior>
