In `src/pipeline/premise/parser.py`:

Replace the current `PREMISE_BLOCK_RE` (4-line pattern ending at SCOPE) with a 5-group pattern that adds an optional GENUS line:

```python
PREMISE_BLOCK_RE = re.compile(
    r"^\s*PREMISE:\s*(.+?)[ \t]*(?:\r?\n)"
    r"^\s*VALIDATED_BY:\s*(.+?)[ \t]*(?:\r?\n)"
    r"^\s*FOIL:\s*(.+?)[ \t]*(?:\r?\n)"
    r"^\s*SCOPE:\s*(.+?)[ \t]*(?:\r?\n|$)"
    r"(?:^\s*GENUS:\s*(.+?)[ \t]*(?:\r?\n|$))?",
    re.MULTILINE,
)
```

Key constraints:
- The SCOPE line must remain non-greedy (`.+?`) — do NOT change it to greedy or it will consume the GENUS line
- The GENUS group (group 5) is wrapped in `(?:...)?` making it optional — this is what preserves all existing tests
- The `(?:\r?\n|$)` on the SCOPE line must allow end-of-string so a 4-line block at end of text still matches
- Group ordering is unchanged for groups 1-4: claim, validated_by, foil_line, scope

In `parse_premise_blocks()`, after extracting scope (group 4), extract the GENUS group and parse it:

```python
genus_line = match.group(5)
genus_name = None
genus_instances: list[str] | None = None
if genus_line:
    genus_line = genus_line.strip()
    # Format: "name | INSTANCES: [A, B]" or "name | INSTANCES: A, B"
    if " | INSTANCES: " in genus_line:
        parts = genus_line.split(" | INSTANCES: ", 1)
        genus_name = parts[0].strip()
        raw_instances = parts[1].strip()
        # Strip brackets if present: "[A, B]" -> "A, B"
        raw_instances = raw_instances.strip("[]")
        genus_instances = [i.strip() for i in raw_instances.split(",") if i.strip()]
    else:
        genus_name = genus_line  # No INSTANCES sub-field
```

Pass `genus_name` and `genus_instances` to the `ParsedPremise` constructor.

Both are optional with None defaults — this is the key backward-compatibility guarantee. All existing `ParsedPremise` construction sites pass no genus args and will continue to work unchanged.

Do NOT add genus fields to `PremiseRecord` — the research explicitly prohibits this to avoid schema churn. Genus data flows to `axis_edges`, not into `premise_registry`.
