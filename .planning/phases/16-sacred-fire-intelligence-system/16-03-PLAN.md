**After the existing profile display (lines 72-80), add TE section:**

Query transport_efficiency_sessions for the subject:
```sql
-- For human profiles:
SELECT raven_depth, crow_efficiency, transport_speed, trunk_quality,
       composite_te, trunk_quality_status, fringe_drift_rate
FROM transport_efficiency_sessions
WHERE human_id = ? AND subject = 'human'
ORDER BY created_at DESC
LIMIT 10
```

```sql
-- For AI profiles:
SELECT raven_depth, crow_efficiency, transport_speed, trunk_quality,
       composite_te, trunk_quality_status, fringe_drift_rate
FROM transport_efficiency_sessions
WHERE subject = 'ai'
ORDER BY created_at DESC
LIMIT 10
```

**Display format (human profile):**
```
Intelligence Profile: {human_id}
========================================
Sessions:         {session_count}
Flame Frequency:  {flame_frequency}
Avg Marker Level: {avg_marker_level:.1f}
Max Marker Level: {max_marker_level}
Spiral Depth:     {spiral_depth}
Flood Rate:       {flood_rate:.2f} ({pct}%)

TransportEfficiency (last session):
  Raven Depth:      {raven_depth:.3f}
  Crow Efficiency:  {crow_efficiency:.3f}
  Transport Speed:  {transport_speed:.3f}
  Trunk Quality:    {trunk_quality:.3f} ({trunk_quality_status})
  Composite TE:     {composite_te:.4f}
  Fringe Drift:     {fringe_drift_rate:.1f} (avg recent: {avg_fringe:.2f})

TE Trend (last 10 sessions):
  {composite_te_1:.4f}  {composite_te_2:.4f}  ... (N pending, M confirmed)
```

For AI profile (`show_ai=True` or `human_id == "ai"`), also show te_delta ranking:

```sql
SELECT mc.ccd_axis, mc.te_delta, mc.pre_te_avg, mc.post_te_avg
FROM memory_candidates mc
WHERE mc.status = 'validated' AND mc.te_delta IS NOT NULL
ORDER BY mc.te_delta DESC
```

**AI profile additional display:**
```
Top Memory Entries by TE Impact:
  1. {ccd_axis} (+{te_delta:.4f} TE delta)
  2. {ccd_axis} (+{te_delta:.4f} TE delta)
  ...
  [{N} entries pending backfill]
```

Count pending backfill: `SELECT COUNT(*) FROM memory_candidates WHERE status='validated' AND te_delta IS NULL`

**Pending vs confirmed breakdown:**
Count pending: `SELECT COUNT(*) FROM transport_efficiency_sessions WHERE subject=? AND trunk_quality_status='pending'`
Count confirmed: same with 'confirmed'.
Display: "(N pending, M confirmed)" after TE trend.

**Wrap all TE queries in try/except** -- if transport_efficiency_sessions doesn't exist (database from before Phase 16), gracefully skip TE display with "TransportEfficiency: not yet computed"

**Do NOT modify the IntelligenceProfile model** (per Q7 locked decision -- existing 6 metrics stay as-is). TE display is pure CLI formatting from SQL queries.
