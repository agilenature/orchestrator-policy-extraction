<objective>
Create the Premise Registry data layer: DuckDB schema for `premise_registry` table, Pydantic models for premise records, PREMISE block regex parser, DuckDB CRUD operations, and `parent_episode_id` causal links on the episodes table with backfill.

Purpose: This plan establishes the data foundation for the Premise-Assertion Gate. Without the registry table and CRUD, the PAG hook (Plan 02) has nowhere to store or query premises. Without `parent_episode_id`, the episode causal graph (PREMISE-05) cannot be traversed.

Output: New `src/pipeline/premise/` module with models, parser, registry, schema. Updated `src/pipeline/storage/schema.py` and `src/pipeline/runner.py`.
</objective>

    **`src/pipeline/premise/registry.py`**: PremiseRegistry class with DuckDB CRUD.

    `class PremiseRegistry`:
    - `__init__(self, conn: duckdb.DuckDBPyConnection)`: Store connection. Do NOT call create_premise_schema here (caller's responsibility, matching project pattern where `create_schema()` is called once in runner init).
    - `register(self, record: PremiseRecord) -> str`: INSERT OR REPLACE into premise_registry. Return premise_id. Use parameterized query with `json.dumps()` for JSON columns.
    - `get(self, premise_id: str) -> PremiseRecord | None`: SELECT by premise_id, return PremiseRecord or None.
    - `get_by_session(self, session_id: str) -> list[PremiseRecord]`: SELECT WHERE session_id = ?, ordered by created_at.
    - `get_stained(self, project_scope: str | None = None) -> list[PremiseRecord]`: SELECT WHERE staining_record->>'stained' = 'true', optionally filtered by project_scope.
    - `update_staleness(self, premise_id: str) -> None`: INCREMENT staleness_counter by 1, update updated_at.
    - `stain(self, premise_id: str, stained_by: str, ground_truth_pointer: dict) -> None`: UPDATE staining_record JSON to `{"stained": true, "stained_by": stained_by, "stained_at": now_iso, "ground_truth_pointer": ground_truth_pointer}`, update updated_at.
    - `find_by_foil(self, foil_text: str, project_scope: str | None = None, exclude_session: str | None = None, limit: int = 10) -> list[PremiseRecord]`: SELECT WHERE claim ILIKE '%foil_text%' with optional project_scope filter and session exclusion. Order by created_at DESC. This is the foil instantiation lookup (used by Plan 03).
    - `count(self) -> int`: SELECT count(*).
    - `backfill_parent_episodes(self) -> int`: Execute PARENT_EPISODE_BACKFILL_SQL, return rows affected.

    All methods use parameterized queries (no f-string SQL). JSON columns are inserted as `json.dumps(val)` strings.
