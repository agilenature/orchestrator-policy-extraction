Create `tests/test_structural_integration.py` with ~18 integration tests organized by BRIDGE requirement number (following the TestDDF01/TestDDF02 pattern from tests/test_ddf_integration.py).

**Test fixture setup** (shared conftest-style fixture at module level or class setUp):
- In-memory DuckDB connection.
- Call `create_ddf_schema(conn)` to create all tables including structural_events.
- Helper function `_insert_flame_event(conn, **kwargs)` that INSERTs a row into flame_events with sensible defaults (session_id='test-session', subject='human', detection_source='stub', etc.).
- Helper function `_insert_axis_edge(conn, axis_a, axis_b, **kwargs)` for dependency sequencing tests.

**class TestBRIDGE01_StructuralEventDetection** (8 tests):

1. `test_all_four_signal_types_produced` -- Insert flame_events covering all four signal types for a session. Call detect_structural_signals(). Verify output contains events with signal_type in ('gravity_check', 'main_cable', 'dependency_sequencing', 'spiral_reinforcement'). At minimum gravity_check and main_cable should be present.

2. `test_gravity_check_both_outcomes` -- Insert one L5 with grounding (L1 same axis within 3 prompts) and one L5 without grounding. Verify two gravity_check events: one signal_passed=True, one signal_passed=False.

3. `test_main_cable_with_evidence_chain` -- Insert L5 events for an axis from 2+ distinct source_episode_ids. Verify main_cable signal_passed=True with contributing_flame_event_ids populated.

4. `test_dependency_sequencing_respected` -- Insert axis_edge (axis_a, axis_b). Insert L3+ for axis_a at prompt 1, L5+ for axis_b at prompt 5. Verify dependency_sequencing signal_passed=True.

5. `test_dependency_sequencing_violated` -- Insert axis_edge (axis_a, axis_b). Insert L5+ for axis_b at prompt 1 WITHOUT prior L3+ for axis_a. Verify dependency_sequencing signal_passed=False.

6. `test_spiral_reinforcement_from_floods` -- Insert L6 flood_confirmed=True events. Verify spiral_reinforcement signal_type events emitted with signal_passed=True.

7. `test_human_and_ai_subjects_detected` -- Insert flame_events for both human and ai subjects. Verify structural_events produced for both subjects.

8. `test_structural_events_written_to_db` -- Call detect + write. Query structural_events table. Verify rows exist with correct schema (event_id, signal_type, signal_passed, contributing_flame_event_ids as array).

**class TestBRIDGE02_StructuralIntegrityScore** (5 tests):

9. `test_score_computed_for_session` -- Insert flame_events and structural_events. Call compute_structural_integrity(). Verify result is a StructuralIntegrityResult with score in [0.0, 1.0].

10. `test_score_null_when_no_l5_events` -- Session with only L0-L2 events. Verify compute_structural_integrity returns None.

11. `test_score_formula_correct` -- Insert known structural_events and flame_events. Compute score manually using the ASYMMETRIC DENOMINATOR formula:
   - gravity_ratio = gravity_pass_count / total_L5plus_flame_events (denominator from flame_events table, NOT structural_events)
   - main_cable_ratio = main_cable_pass_count / main_cable_structural_total (denominator from structural_events)
   - dependency_ratio = dependency_pass_count / dependency_structural_total (denominator from structural_events)
   - spiral_capped = min(spiral_count, 3) / 3.0
   Verify the computed score matches the expected value within 0.001.

12. `test_score_neutral_fallback` -- Session with L5+ events but NO main cable events at all. main_cable_ratio should be 0.5 (neutral fallback). Verify score reflects this.

13. `test_score_per_subject` -- Compute for human and AI separately. Both should return non-None when both have L5+ events. Scores may differ.

**class TestBRIDGE03_Op8CorrectionDeposit** (5 tests):

14. `test_op8_end_to_end_deposit` -- Insert AI L5+ flame_event with no L0-L2 grounding and flood_confirmed=False. Call deposit_op8_corrections(). Verify memory_candidates row with source_type='op8_correction', ccd_axis matching, fidelity=2, confidence=0.60, subject='ai'.

15. `test_op8_one_per_axis_dedup` -- Insert 3 AI L5+ events for same axis. deposit_op8_corrections produces exactly 1 correction. detection_count = 3.

16. `test_op8_skips_when_grounded` -- Insert AI L5+ AND AI L1 for same axis. deposit_op8_corrections produces 0 corrections.

17. `test_op8_skips_flood_confirmed` -- Insert AI L5+ with flood_confirmed=True (even without grounding). deposit_op8_corrections produces 0 corrections (condition b: flood_confirmed=False required).

18. `test_op8_correction_appears_in_memory_review` -- After deposit, query memory_candidates WHERE source_type='op8_correction' AND status='pending'. Verify it would appear in the memory-review CLI queue.

All tests use in-memory DuckDB. Each test class sets up its own fresh database to avoid cross-test contamination.
