<objective>
Build the Premise-Assertion Gate (PAG) PreToolUse hook: transcript backward scanner, PREMISE block extraction, staging writer, and hook script that validates premises at write-class tool call boundaries.

Purpose: This is the introspective layer -- the component that intercepts the AI's tool calls in real-time and validates explicit PREMISE declarations. Without this hook, premises exist only in conversation text (invisible to the system). The hook makes them machine-readable and actionable.

Output: New `src/pipeline/live/hooks/premise_gate.py` hook script, `src/pipeline/premise/transcript.py` scanner, `src/pipeline/premise/staging.py` writer.
</objective>

    **`src/pipeline/premise/transcript.py`**: Backward JSONL scanner.

    `read_recent_assistant_text(transcript_path: str, max_lines: int = 50) -> list[str]`:
    - Read backward from EOF using seek (see RESEARCH.md code example). Read last `max_lines` lines.
    - Parse each line as JSON. Filter for entries where `obj.get("type") == "assistant"`.
    - For each assistant entry, extract text from `obj["message"]["content"]` where content item `type == "text"`.
    - Return texts in chronological order (oldest first from the scanned window).
    - Handle gracefully: file not found (return []), empty file (return []), non-JSON lines (skip), missing content field (skip).
    - Performance target: <5ms for 50 lines of a 14MB file.

    `count_validation_calls_since_last_user(transcript_path: str, max_lines: int = 200) -> int`:
    - Read backward from EOF (larger window than text scanner -- validation calls may be spread across many lines).
    - Scan lines in reverse chronological order.
    - Count assistant entries with `tool_use` content where `name` is in `{"Read", "Grep", "Glob", "WebFetch"}`.
    - Stop counting when a `user` type entry is encountered (this is the boundary).
    - Return the count. This implements the `validation_calls_before_claim` field for Ad Ignorantiam detection (RQR=0).
    - If no user entry found, return the total count of validation calls in the window.

    **`src/pipeline/premise/staging.py`**: Append-only JSONL staging writer.

    This is the solution to the DuckDB two-writer conflict (RESEARCH.md Pitfall 4, Open Question 3). The PAG hook writes to a JSONL file; the batch pipeline ingests into ope.db.

    `STAGING_PATH = "data/premise_staging.jsonl"` (default, overridable)

    `append_to_staging(records: list[dict], staging_path: str = STAGING_PATH) -> int`:
    - Append records to the staging JSONL file (one JSON object per line).
    - Create parent directory if needed.
    - Use file locking (`fcntl.flock` on macOS/Linux) to handle concurrent hook invocations.
    - Each record dict should contain all PremiseRecord fields.
    - Return number of records written.

    `read_staging(staging_path: str = STAGING_PATH) -> list[dict]`:
    - Read all records from the staging file.
    - Return list of dicts. Handle: file not found (return []), corrupt lines (skip with log warning).

    `clear_staging(staging_path: str = STAGING_PATH) -> int`:
    - Read count, then truncate the file. Return count of records cleared.
    - This is called by the batch pipeline AFTER successful ingestion.

    Create the `src/pipeline/live/` and `src/pipeline/live/hooks/` directories with `__init__.py` files (empty, for package structure).

    **`src/pipeline/live/hooks/premise_gate.py`**: The PAG PreToolUse hook.

    This script is invoked by Claude Code via the PreToolUse hook protocol. It receives JSON on stdin and outputs JSON on stdout. Reference the existing hook at `~/.claude/hooks/redirect-to-perplexity.sh` for the protocol pattern, but implement in Python for access to the premise module.

    **Hook stdin JSON fields** (from Claude Code hooks reference):
    ```json
    {
      "session_id": "...",
      "transcript_path": "...",
      "cwd": "...",
      "tool_name": "Edit|Write|Bash|Read|...",
      "tool_input": {...}
    }
    ```

    **Implementation (`def main()`):**

    1. Read stdin as JSON. Extract `tool_name`, `transcript_path`, `session_id`, `tool_input`, `cwd`.

    2. **Gate check**: If `tool_name` not in `WRITE_CLASS = {"Edit", "Write", "Bash"}`, exit 0 immediately (validation-class tool, no PREMISE needed).

    3. **Read transcript**: Call `read_recent_assistant_text(transcript_path)` to get recent assistant text blocks.

    4. **Parse PREMISE blocks**: Call `parse_premise_blocks(text)` for each text block. Collect all `ParsedPremise` objects.

    5. **Handle no PREMISE found**: If no PREMISE blocks found AND the tool is write-class, this could mean:
       - (a) Trivial change not requiring premises (common case)
       - (b) Text not yet flushed to JSONL (Pitfall 1)
       - (c) AI forgot to declare premises
       Log a debug message and exit 0 (fail-open for Phase 14.1).

    6. **Count validation calls**: Call `count_validation_calls_since_last_user(transcript_path)` to get the `validation_calls_before_claim` count.

    7. **Build staging records**: For each ParsedPremise, create a dict with all PremiseRecord fields:
       - `premise_id`: PremiseRecord.make_id(claim, session_id, tool_use_id) -- use `tool_input.get("id", "")` or generate from tool_name + timestamp as tool_use_id
       - `claim`, `validated_by`, `foil`, `distinguishing_prop`, `scope` from parsed block
       - `session_id` from hook input
       - `project_scope` = cwd
       - `validation_calls_before_claim` from counter
       - `derivation_depth` = 0 (default, updated by batch pipeline)
       - `staining_record` = None (set by batch pipeline)
       - All other fields = None/default

    8. **Write to staging**: Call `append_to_staging(records)`.

    9. **Check for warnings**: Build `additionalContext` messages for:
       - **UNVALIDATED on high-risk file**: If any premise `is_unvalidated` AND `tool_input` targets a high-risk path (defined as paths matching: `src/pipeline/storage/schema.py`, `data/constraints.json`, `.claude/settings.json`, `src/pipeline/models/`), add warning: `"PREMISE_UNVALIDATED_HIGH_RISK: Premise '{claim}' is UNVALIDATED and targets a high-risk file. Consider validating with a Read/Grep call first."`
       - **Stained premise detected**: If `data/ope.db` exists, open a READ-ONLY DuckDB connection (`duckdb.connect(db_path, read_only=True)`), check if any of the current premises' claims match a stained premise in the registry (ILIKE match on claim text). If found, add: `"PROJECTION_WARNING: Premise '{claim}' was previously stained (invalidated by {stained_by}). Historical foil evidence: {foil_path_outcomes}."` Close the read-only connection. Wrap entire staining check in try/except (fail-open if ope.db is locked or missing).

    10. **Emit response**: If there are additionalContext messages, output JSON:
        ```json
        {
          "hookSpecificOutput": {
            "hookEventName": "PreToolUse",
            "additionalContext": "...(joined messages)..."
          }
        }
        ```
        Exit 0 (allow tool call with context). If no messages, exit 0 silently.

    **Important constraints:**
    - The hook MUST NOT block any tool call in Phase 14.1 (exit code always 0, never 2). Warnings only via additionalContext.
    - The hook MUST NOT write to `data/ope.db` (two-writer conflict). All writes go to `data/premise_staging.jsonl`.
    - Read-only DuckDB connection for staining check MUST be wrapped in try/except.
    - Total hook latency target: <50ms (backward scan <5ms, parse <1ms, staging write <5ms, staining check <30ms).

    Add `if __name__ == "__main__": main()` so the hook can be invoked as `python src/pipeline/live/hooks/premise_gate.py`.

    **Tests** (`test_premise_gate.py`):
    - Mock stdin with JSON payloads for different tool_name values.
    - Test that validation-class tools (Read, Grep) cause immediate exit 0 with no output.
    - Test that write-class tools with PREMISE blocks in transcript produce staging records.
    - Test UNVALIDATED warning on high-risk file paths.
    - Test that staining check works with a pre-populated in-memory DuckDB.
    - Test fail-open when transcript is missing or empty.
    - Test additionalContext output format matches Claude Code hook protocol.
    - Use monkeypatch to replace stdin and staging paths.

    Also create empty `tests/pipeline/live/__init__.py` and `tests/pipeline/live/hooks/__init__.py`.
