## Decisions Made
- Adapted plan's schema references to match existing memory_candidates table (uses `id` not `candidate_id`, `status` not `verdict`, no `epistemological_origin` column) -- the existing schema from plan 13.3-01 is the ground truth
- Rejected verdicts without opinions still record in TrustAccumulator (reject_count increments) -- ensures trust state reflects all review activity, not just those with opinions
- Used DuckDB `ON CONFLICT DO NOTHING` syntax instead of plan's `INSERT OR IGNORE` (SQLite syntax incompatible with DuckDB)
- `heuristic_description` field populated with "Reactive spec-correction from rejected verdict on {point_id}" to provide context on candidate origin
