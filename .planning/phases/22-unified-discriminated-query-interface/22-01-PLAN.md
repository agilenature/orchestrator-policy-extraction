```python
def query_sessions(
    query: str,
    db_path: str = "data/ope.db",
    top_n: int = 5,
) -> list[dict[str, Any]]:
```

Algorithm:
1. Connect to DuckDB at `db_path` (NOT read_only -- FTS extension needs it).
2. Check if FTS index exists: `SELECT table_name FROM duckdb_tables() WHERE table_name LIKE 'fts_main_episode_search_text%'`. If found, load FTS extension via `LOAD fts;`.
3. If FTS index exists, run BM25 search (exact pattern from `HybridRetriever._bm25_search()` in `src/pipeline/rag/retriever.py` lines 82-98):
   ```sql
   SELECT sq.episode_id, sq.score
   FROM (
       SELECT *, fts_main_episode_search_text.match_bm25(
           episode_id, ?
       ) AS score
       FROM episode_search_text
   ) sq
   WHERE sq.score IS NOT NULL
   ORDER BY sq.score DESC
   LIMIT ?
   ```
4. If FTS index does NOT exist, fall back to ILIKE:
   ```sql
   SELECT episode_id, 1.0 AS score
   FROM episode_search_text
   WHERE search_text ILIKE ?
   LIMIT ?
   ```
   Pass `f"%{query}%"` as the ILIKE parameter.
5. If no results from either path, check if `episode_search_text` table has any rows. If 0 rows, log a warning suggesting `python -m src.pipeline.cli train embed`.
6. For matched episode_ids, enrich with metadata from `episodes` table:
   ```sql
   SELECT e.episode_id, e.session_id, e.mode,
          LEFT(est.search_text, 200) as preview
   FROM episodes e
   JOIN episode_search_text est ON e.episode_id = est.episode_id
   WHERE e.episode_id IN (...)
   ```
7. Return list of dicts with keys: `source` ("sessions"), `episode_id`, `session_id`, `content_preview`, `match_reason` (e.g. "bm25 (score=-1.23)" or "ilike").
8. Entire function wrapped in try/except returning [] on any error (fail-open pattern matching `query_docs()`).

Use `conn.close()` in a `finally` block (not context manager -- matching `query_docs()` pattern).

Create `src/pipeline/code_query.py` with a single public function:

```python
def query_code(
    query: str,
    search_dir: str = "src/",
    top_n: int = 10,
) -> list[dict[str, Any]]:
```

Algorithm:
1. Check for ripgrep: `shutil.which("rg")`. If found, build command:
   `[rg_path, "-n", "-i", "--max-count", "3", "--type", "py", "--type", "md", query, search_dir]`
   The `--max-count 3` limits matches per file to avoid flooding from a single file.
2. If rg not found, fall back to grep:
   `["grep", "-rn", "-i", "--include=*.py", "--include=*.md", query, search_dir]`
3. Run via `subprocess.run(cmd, capture_output=True, text=True, timeout=10)`.
4. If returncode not in (0, 1), return []. (1 = no matches for both rg and grep.)
5. Parse stdout line by line. Format is `file:line:content`. Split on ":" with maxsplit=2.
6. For each parsed line, build result dict: `source` ("code"), `file_path`, `line_number` (int), `content_preview` (stripped, truncated to 120 chars), `match_reason` ("text match").
7. Limit to top_n results total.
8. Entire function wrapped in try/except returning [] on any error (fail-open).
