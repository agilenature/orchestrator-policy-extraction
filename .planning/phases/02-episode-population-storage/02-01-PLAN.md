**DuckDB episodes table (extend `storage/schema.py`):**

Add the `episodes` table to `create_schema()` following the hybrid pattern from research:

```sql
CREATE TABLE IF NOT EXISTS episodes (
    -- Identity (flat, queryable)
    episode_id VARCHAR PRIMARY KEY,
    session_id VARCHAR NOT NULL,
    segment_id VARCHAR NOT NULL,
    timestamp TIMESTAMPTZ NOT NULL,

    -- Flat queryable columns (duplicated from nested for fast filtering)
    mode VARCHAR,
    risk VARCHAR,
    reaction_label VARCHAR,
    reaction_confidence FLOAT,
    outcome_type VARCHAR,

    -- STRUCT for typed nested data (queryable via dot notation)
    observation STRUCT(
        repo_state STRUCT(
            changed_files VARCHAR[],
            diff_stat STRUCT(files INTEGER, insertions INTEGER, deletions INTEGER)
        ),
        quality_state STRUCT(
            tests_status VARCHAR,
            lint_status VARCHAR,
            build_status VARCHAR
        ),
        context STRUCT(
            recent_summary VARCHAR,
            open_questions VARCHAR[],
            constraints_in_force VARCHAR[]
        )
    ),

    -- JSON for flexible nested data
    orchestrator_action JSON,
    outcome JSON,
    provenance JSON,
    labels JSON,

    -- Provenance flat columns
    source_files VARCHAR[],
    config_hash VARCHAR,

    -- Metadata
    schema_version INTEGER DEFAULT 1,
    created_at TIMESTAMPTZ DEFAULT current_timestamp,
    updated_at TIMESTAMPTZ DEFAULT current_timestamp
);
```

Add indexes: `idx_episodes_session(session_id)`, `idx_episodes_mode(mode)`, `idx_episodes_risk(risk)`, `idx_episodes_reaction(reaction_label)`, `idx_episodes_ts(timestamp)`.

Update `drop_schema()` to drop `episodes` table (before episode_segments, respecting order).

**EpisodeValidator (`src/pipeline/episode_validator.py`):**

Create class that:
1. Loads `data/schemas/orchestrator-episode.schema.json` via `jsonschema` library
2. Uses `jsonschema.validators.validator_for()` to auto-detect Draft 2020-12
3. Creates a `FormatChecker` for date-time validation
4. `validate(episode_dict) -> tuple[bool, list[str]]` -- returns (is_valid, error_messages)
5. Additional checks beyond schema: provenance must have >=1 source, confidence in [0,1], mode is valid enum
6. `validate_batch(episodes) -> dict` -- returns stats: {valid: N, invalid: N, errors: [...]}
