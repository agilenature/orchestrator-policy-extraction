Class `AssessmentSessionRunner`:
```python
class AssessmentSessionRunner:
    """Manages the full lifecycle of an assessment session.

    Lifecycle:
    1. setup_assessment_dir: Create /tmp dir, write scenario files, pre-seed CLAUDE.md + MEMORY.md
    2. launch_actor: Launch Claude Code subprocess in assessment dir
    3. cleanup_session: Tar assessment dir, store path, rm -rf

    Args:
        conn: DuckDB connection for reading scenarios and writing results.
        db_path: Path to DuckDB file (for Observer pipeline).
    """

    def __init__(self, conn: duckdb.DuckDBPyConnection, db_path: str = "data/ope.db") -> None:
        self._conn = conn
        self._db_path = db_path
```

Method `setup_assessment_dir(self, scenario_spec: ScenarioSpec, session_id: str) -> AssessmentSession`:
1. Create dir: `/tmp/ope_assess_{session_id}/`
2. Write scenario files using ScenarioGenerator.generate_scenario_files()
3. Pre-seed CLAUDE.md:
   - If scenario has handicap_claude_md: write that to `{assessment_dir}/.claude/CLAUDE.md`
   - If no handicap: write a minimal assessment CLAUDE.md:
     ```
     # Assessment Session
     You are participating in a collaborative coding session.
     Focus on helping the user diagnose and fix the issue in the implementation file.
     ```
4. Pre-seed MEMORY.md with current production IntelligenceProfile:
   - Path: `~/.claude/projects/{encoded_dir}/memory/MEMORY.md`
   - encoded_dir = assessment_dir with slashes replaced by dashes
   - Create the directory: `os.makedirs(memory_dir, exist_ok=True)`
   - Read the production MEMORY.md from `_DEFAULT_MEMORY_PATH` (from intelligence.py) or a configurable path
   - Copy its content to the assessment MEMORY.md path
5. Return `AssessmentSession` with status="setup", all paths populated

Method `launch_actor(self, session: AssessmentSession, prompt: str | None = None) -> AssessmentSession`:
1. Build command:
   ```python
   # The prompt instructs the AI to help with the scenario
   default_prompt = (
       "Read scenario_context.md to understand the problem, "
       "then diagnose and fix the broken implementation file."
   )
   cmd_prompt = prompt or default_prompt
   # Use -p flag for non-interactive prompt-based session
   cmd = f'cd "{session.assessment_dir}" && unset CLAUDECODE && claude -p "{cmd_prompt}" --session-id {session.session_id}'
   ```
2. Launch via subprocess.run with shell=True, capture_output=True, timeout configurable (default 1800 = 30 min)
3. Derive JSONL path: `AssessmentSession.derive_jsonl_path(session.assessment_dir, session.session_id)`
4. Return updated session with status="completed" (or "failed" if subprocess error), jsonl_path populated

Method `cleanup_session(self, session: AssessmentSession) -> AssessmentSession`:
1. Tar the assessment dir: `tarfile.open(tar_path, "w:gz")` where tar_path = `/tmp/ope_assess_{session_id}.tar.gz`
2. Store tar path in assessment_te_sessions: `UPDATE assessment_te_sessions SET session_artifact_path = ? WHERE session_id = ?`
3. Remove assessment dir: `shutil.rmtree(session.assessment_dir, ignore_errors=True)`
4. Also clean up the project-specific Claude files: `~/.claude/projects/{encoded_dir}/` (rm -rf)
5. Return updated session with session_artifact_path set

Convenience functions:
```python
def setup_assessment_dir(conn, scenario_spec, session_id) -> AssessmentSession:
    return AssessmentSessionRunner(conn).setup_assessment_dir(scenario_spec, session_id)

def launch_actor(conn, session, prompt=None) -> AssessmentSession:
    return AssessmentSessionRunner(conn).launch_actor(session, prompt)

def cleanup_session(conn, session) -> AssessmentSession:
    return AssessmentSessionRunner(conn).cleanup_session(session)
```

```python
def compute_assessment_te(
    conn: duckdb.DuckDBPyConnection,
    session_id: str,
) -> dict | None:
    """Compute 3-metric assessment TE (no transport_speed).

    Assessment TE = raven_depth * crow_efficiency * trunk_quality

    Per locked decision Q6: transport_speed excluded from assessment formula.
    Assessment sessions are too short for meaningful transport speed measurement.

    Returns dict with: raven_depth, crow_efficiency, trunk_quality, candidate_te.
    Returns None if no flame_events for session.
    """
```

Implementation:
1. Query flame_events for the session (same SQL as compute_te_for_session but without transport_speed):
   ```sql
   SELECT
       MAX(marker_level) / 7.0 AS raven_depth,
       CAST(COUNT(*) FILTER (WHERE axis_identified IS NOT NULL) AS FLOAT)
           / NULLIF(COUNT(*), 0) AS crow_efficiency,
       0.5 AS trunk_quality  -- pending, same as production
   FROM flame_events
   WHERE session_id = ? AND subject = 'human'
   ```
2. Compute: `candidate_te = raven_depth * crow_efficiency * trunk_quality`
3. Return dict or None if no rows.

```python
def compute_candidate_ratio(
    candidate_te: float,
    scenario_baseline_te: float,
) -> float | None:
    """Compute candidate_ratio = candidate_te / scenario_baseline_te.

    Returns None if scenario_baseline_te is 0 or None.

    if not scenario_baseline_te or scenario_baseline_te == 0:

    return candidate_te / scenario_baseline_te
```

```python
class AssessmentObserver:
    """Post-session analysis of assessment JSONL via existing OPE pipeline.

    Runs PipelineRunner.run_session() on the assessment transcript,
    then tags all generated flame_events with assessment_session_id.

    Does NOT hand-roll JSONL parsing -- reuses the full OPE pipeline.

    Args:
        db_path: Path to DuckDB database.
        config_path: Path to pipeline config YAML.
    """

    def __init__(
        self,
        db_path: str = "data/ope.db",
        config_path: str = "data/config.yaml",
    ) -> None:
        self._db_path = db_path
        self._config_path = config_path
```

Method `run_observation(self, session: AssessmentSession) -> dict`:
1. Verify JSONL file exists at session.jsonl_path. If not, raise FileNotFoundError.
2. Load config: `config = load_config(self._config_path)`
3. Create PipelineRunner: `runner = PipelineRunner(config, db_path=self._db_path)`
4. Run session: `stats = runner.run_session(session.jsonl_path)` -- this processes JSONL through the full pipeline (tagging, segmentation, population, DDF detection, TE computation, etc.)
5. After pipeline run, tag assessment flame_events:
   ```python
   conn = duckdb.connect(self._db_path)
   # Tag all flame_events from this session with assessment_session_id
   conn.execute(
       "UPDATE flame_events SET assessment_session_id = ? WHERE session_id = ?",
       [session.session_id, session.session_id],
   )
   conn.close()
   ```
6. Return stats dict from pipeline run.

Convenience function:
```python
def run_observer(session: AssessmentSession, db_path: str = "data/ope.db") -> dict:
    observer = AssessmentObserver(db_path=db_path)
    return observer.run_observation(session)
```

Class `RejectionDetector`:
```python
class RejectionDetector:
    """Outcome-gated Level 5-7 rejection detection.

    Per locked decision Q5:
    - Rejection = Level 5 ONLY IF candidate_te > scenario_baseline_te * 0.9
    - Rejection + candidate_te < threshold -> stubbornness_indicator=True (not L5)
    - Fringe-signal rejections bypass outcome gate (count as L5 status=pre_naming)
    - transport_speed excluded from assessment TE formula

    Args:
        conn: DuckDB connection.
    """

    def __init__(self, conn: duckdb.DuckDBPyConnection) -> None:
        self._conn = conn
```

Method `detect_rejections(self, session_id: str, scenario_baseline_te: float) -> list[dict]`:
1. Query flame_events for the session to find rejection candidates:
   - Look for human flame_events where the candidate diverged from AI suggestion
   - Heuristic: human event following an AI event, where human's tool call differs from AI's suggested action
   - SQL: query flame_events for session, ordered by prompt_number, look for patterns where subject='human' marker_type contains 'correction' or 'question' followed by divergent action
2. For each rejection candidate:
   a. Check if it's a fringe-signal rejection:
      - marker_type contains 'fringe' or marker_level in (1, 2) with evidence containing fringe-like language ("something feels off", "doesn't sit right")
      - If fringe: immediately classify as L5 with `status='pre_naming'`, bypass outcome gate
   b. Apply outcome gate:
      - Compute candidate_te for the session using compute_assessment_te()
      - If candidate_te > scenario_baseline_te * 0.9: Level 5 rejection confirmed
      - If candidate_te <= scenario_baseline_te * 0.9: stubbornness_indicator=True, NOT Level 5
3. Return list of dicts: `[{"prompt_number": N, "rejection_type": "L5"|"stubbornness"|"fringe_L5", "candidate_te": X, "threshold": Y, "evidence": "..."}]`

Method `classify_rejection(self, candidate_te: float | None, scenario_baseline_te: float, is_fringe: bool) -> str`:
```python
if is_fringe:
    return "fringe_L5"  # Bypass outcome gate
if candidate_te is None:
    return "stubbornness"  # Can't compute -- conservative
threshold = scenario_baseline_te * 0.9
if candidate_te > threshold:
    return "L5"  # Outcome-gated confirmation
return "stubbornness"  # Failed outcome gate
```

Convenience function:
```python
def detect_rejections(conn, session_id: str, scenario_baseline_te: float) -> list[dict]:
    return RejectionDetector(conn).detect_rejections(session_id, scenario_baseline_te)
```
