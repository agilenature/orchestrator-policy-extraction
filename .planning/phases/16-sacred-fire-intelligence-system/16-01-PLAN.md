  <action>
Add `memory-review` subcommand to the intelligence group in `src/pipeline/cli/intelligence.py`.

**CLI command:** `python -m src.pipeline.cli intelligence memory-review`

**Options:**
- `--db` (default "data/ope.db") -- DuckDB database path
- `--memory-file` (default None) -- override MEMORY.md path; if None, derive from project: `~/.claude/projects/-Users-david-projects-orchestrator-policy-extraction/memory/MEMORY.md`

**Implementation (per Q4 locked decision):**

1. Open DuckDB connection (read-write, not read-only).
2. Query: `SELECT id, ccd_axis, scope_rule, flood_example, confidence, subject, source_flame_event_id, session_id, detection_count FROM memory_candidates WHERE status='pending' ORDER BY confidence DESC, detection_count DESC`
3. If no pending candidates, print "No pending memory candidates." and exit 0.
4. For each candidate, display formatted output:
   ```
   ---
   CANDIDATE [n/total] -- Confidence: {confidence} | Subject: {subject} | Detections: {detection_count}
   ---
   CCD axis:      {ccd_axis}
   Scope rule:    {scope_rule}
   Flood example: {flood_example}
   Source:        flame_event {source_flame_event_id} | session {session_id}
   ---
   [a]ccept  [r]eject  [e]dit  [s]kip  [q]uit
   ```
5. Read single character input via `click.getchar()` or `input()`.

**Accept flow:**
  a. Determine MEMORY.md path (--memory-file or default project path)
  b. Read full MEMORY.md content
  c. Check dedup: case-insensitive substring match of ccd_axis in MEMORY.md text
  d. If found: print WARNING, ask `Proceed anyway? [y/N]`; if N, skip
  e. Format entry as:
     ```
     ---\n\n## {ccd_axis}\n\n**CCD axis:** {ccd_axis}\n**Scope rule:** {scope_rule}\n**Flood example:** {flood_example}\n\n
     ```
  f. Append to MEMORY.md: read full file, append entry, write full file (atomic, not stream append)
  g. UPDATE memory_candidates SET status='validated', reviewed_at=NOW() WHERE id=?
     Note: use 'validated' not 'accepted' because the CHECK constraint is ('pending','validated','suspended','rejected','split_required')

**Reject flow:**
  - UPDATE memory_candidates SET status='rejected', reviewed_at=NOW() WHERE id=?

**Edit flow:**
  a. Write ccd_axis, scope_rule, flood_example to a temp file
  b. Open with $EDITOR (os.environ.get("EDITOR", "vi"))
  c. Re-read temp file on close, parse fields back
  d. Update in-memory candidate with edited values
  e. Then proceed to accept confirmation (show updated candidate, ask a/r/s)

**Skip:** no DB change, advance to next.

**Quit:** exit 0 (all previous accept/reject already committed via individual UPDATE statements).

**Injectable input function:** Accept an `input_fn` parameter (default=`input`) for test isolation, same pattern as `src/pipeline/review/collector.py`.

- Test memory-review CLI skip flow
