**In `src/pipeline/live/bus/server.py`:**

1. Add import at top: `from ..genus_oracle import GenusOracleHandler`

2. In `create_app()`, after `_daemon = daemon if daemon ...` line, add:
   ```python
   _genus_oracle = GenusOracleHandler(conn)
   ```
   The oracle takes the server's existing `conn` (same DuckDB connection, no new connection). This is critical to avoid single-writer conflicts (Pitfall 1 from research).

3. Add the handler function inside `create_app()` (same closure pattern as check, push_link):

   ```python
   async def genus_consult(request: Request) -> JSONResponse:
       """Consult the genus oracle for best-matching genus.

       Given a problem description, searches axis_edges for genus_of
       entries matching the problem via token overlap. Fails open:
       returns null genus on any error.
       """
       try:
           body = await request.json()
           problem = body.get("problem", "")
           repo = body.get("repo", None)
           result = _genus_oracle.query_genus(problem, repo)
           return JSONResponse(result)
       except Exception:
           return JSONResponse({
               "genus": None,
               "instances": [],
               "valid": False,
               "confidence": 0.0,
           })
   ```

4. Add the route to the Starlette routes list:
   ```python
   Route("/api/genus-consult", genus_consult, methods=["POST"]),
   ```

import duckdb
