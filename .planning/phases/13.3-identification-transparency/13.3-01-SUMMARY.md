## Decisions Made
- **UNIQUE constraint on identification_instance_id**: One verdict per instance enforced at DB level (at-most-once semantics)
- **CCD format as CHECK constraint**: The trusted minimal core (is_valid_ccd) is now a schema invariant, not a convention -- empty/whitespace ccd_axis, scope_rule, or flood_example are rejected at INSERT time
- **memory_candidates status enum**: Includes 'pending', 'validated', 'suspended', 'rejected', 'split_required' -- the 'suspended' state was added per the snippet-not-chunk CCD axis requirement
- **Composite natural key instance IDs**: Format `{source_table}:{primary_key}:{point_type}` provides traceable provenance
- **L5 reads from constraints.json**: Constraints are stored as JSON file, not DuckDB -- pool builder adapted accordingly
- **L7 reads from episodes table**: Escalation data stored as columns on episodes, not a separate table
