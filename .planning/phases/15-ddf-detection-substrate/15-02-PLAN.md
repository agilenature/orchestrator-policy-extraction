**markers.py** -- L0, L1, L2 regex detectors. HIGH RECALL, false positives expected (Tier 2 filters). All patterns pre-compiled.

`detect_l0_trunk(text: str) -> tuple[bool, str | None]`:
- Level 0: Human names the core concept/axis. Patterns:
  - `\bthe (?:core|real|actual|fundamental|key|essential|root) (?:issue|problem|question|concept|axis|principle)\b`
  - `\bthis is (?:really|fundamentally|essentially|actually) about\b`
  - `\bthe trunk (?:is|here is)\b`
  - `\bwhat (?:I|we) (?:actually|really) (?:need|want|mean)\b`
  - `\bthe (?:CCD|common denominator|governing axis)\b`
- Returns (detected: bool, evidence_excerpt: str | None with 200 char context window)

`detect_l1_causal(text: str) -> tuple[bool, str | None]`:
- Level 1: Causal language (not just correlation). Patterns:
  - `\b(?:because|since|therefore|caused by|leads to|results in|the reason)\b`
  - `\b(?:if .+ then|when .+ happens)\b`
  - Exclude pure temporal: NOT matching `\bafter\b.*\bthen\b` alone
- Returns (detected, evidence_excerpt)

`detect_l2_assertive(text: str) -> tuple[bool, str | None]`:
- Level 2: Assertive causal (stronger than L1). Patterns:
  - `\b(?:the cause is|this causes|directly leads to|the root cause)\b`
  - `\b(?:I'm certain|I've identified|the answer is)\b`
  - `\b(?:the principle here|the rule is|the invariant)\b`
- Returns (detected, evidence_excerpt)

`detect_markers(events: list[dict], session_id: str) -> list[FlameEvent]`:
- Iterates over events, applies L0/L1/L2 to human_orchestrator user_msg events only
- Extracts text from event["payload"]["common"]["text"] (matching tagger pattern)
- Returns list of FlameEvent with detection_source='stub', subject='human'
- Assigns prompt_number by counting human messages sequentially
- human_id defaults to 'default_human' (per RESEARCH open question 3)

**o_axs.py** -- `OAxsDetector` class:
- Per CONTEXT.md Q3 locked decision:
  - Granularity drop: current_token_count < config.ddf.o_axs.granularity_drop_ratio * avg(prior N prompts)
  - Novel concept: capitalized noun phrase NOT in session known_concepts, appearing 2+ times in last M messages
  - Human messages only
  - known_concepts: per-session reset
- Constructor takes DDFConfig (or just OAxsConfig)
- `reset()` method clears all per-session state (token_counts deque, known_concepts set, recent_messages deque)
- `detect(text: str, actor: str) -> tuple[bool, dict | None]`:
  - Returns False immediately if actor != "human_orchestrator"
  - Tracks token counts in deque(maxlen=prior_prompts_window)
  - Noun phrase regex: `\b([A-Z][a-z]+(?:\s+[A-Z][a-z]+)*)\b`
  - Returns (True, evidence_dict) only when BOTH conditions met
  - evidence_dict: {token_count, avg_prior, drop_ratio, novel_concept}

**writer.py** -- `write_flame_events(conn, events: list[FlameEvent]) -> dict[str, int]`:

**deposit.py** -- `deposit_to_memory_candidates(conn, ccd_axis, scope_rule, flood_example, source_flame_event_id, pipeline_component="ddf_tier2", fidelity=2) -> str | None`:
- Per CONTEXT.md Q8: soft dedup on (ccd_axis, scope_rule) before INSERT
- Normalize before comparison: LOWER(TRIM(ccd_axis)) and LOWER(TRIM(scope_rule))
- If existing match found: UPDATE detection_count = detection_count + 1, return None
- If no match: INSERT new row with deterministic id = SHA-256(ccd_axis + scope_rule + source_flame_event_id)[:16], status='pending', return candidate_id
- Uses the existing memory_candidates table (call create_review_schema + create_ddf_schema if needed in test fixtures)
- Fidelity defaults to 2 per locked decision
- detection_count starts at 1

Also add `mark_deposited(conn, flame_event_id: str) -> None`:
- `UPDATE flame_events SET deposited_to_candidates = TRUE WHERE flame_event_id = ?`
- Called after successful deposit to memory_candidates
