  <behavior>
    The GenusValidator composes five validation layers that each conform to a ValidationLayer Protocol:
    - `validate(episode: dict) -> tuple[bool, list[str]]` returning (is_valid, error_messages)

    **Layer A: SchemaLayer** (wraps existing EpisodeValidator)
    - Valid episode dict -> (True, [])
    - Episode missing required fields -> (False, ["field: error message"])
    - Invalid mode enum -> (False, ["orchestrator_action.mode: must be one of ..."])
    - SHORT-CIRCUITS: if Layer A fails, subsequent layers may still run but GenusValidator collects all errors

    **Layer B: EvidenceGroundingLayer** (warnings, not hard failures)
    - mode=Implement with empty scope.paths -> (True, ["warning:evidence: Implement mode has no scope paths"])
    - mode=Verify with no test/lint results in outcome -> (True, ["warning:evidence: Verify mode has no test results in outcome"])
    - mode=Explore with scope.paths defined -> (True, []) (valid)
    - mode=Integrate with no git_events in outcome -> (True, ["warning:evidence: Integrate mode has no git events"])
    - NOTE: Returns (True, warnings) -- warnings prefixed with "warning:" to distinguish from errors

    **Layer C: NonContradictionLayer** (warnings, not hard failures)
    - mode=Explore + gate write_allowed=true in gates -> (True, ["warning:contradiction: Explore mode with write_allowed gate"])
    - mode=Implement + gate no_write_before_plan without prior plan -> (True, ["warning:contradiction: Implement with no_write_before_plan gate"])
    - mode=Implement with scope.paths -> (True, []) (valid, no contradiction)
    - NOTE: Checks the three specific rules from AUTHORITATIVE_DESIGN.md Part 5.2.C

    **Layer D: ConstraintEnforcementLayer** (severity-aware)
    - Episode scope overlaps with forbidden constraint scope -> (False, ["constraint_enforcement: violates forbidden constraint 'xyz'"])
    - Episode scope overlaps with requires_approval constraint -> (True, ["warning:constraint: requires approval for constraint 'xyz'"])
    - Episode scope overlaps with warning constraint -> (True, ["warning:constraint: warning constraint 'xyz' applies"])
    - No constraint scope overlap -> (True, [])
    - Repo-wide constraint (empty paths) -> applies to all episodes

    **Layer E: EpisodeIntegrityLayer** (hard failures)
    - Missing or empty episode_id -> (False, ["integrity: episode_id is empty or missing"])
    - Missing provenance.sources -> (False, ["integrity: provenance.sources is empty"])
    - reaction.confidence < 0 or > 1 -> (False, ["integrity: reaction confidence out of range"])
    - Valid episode -> (True, [])

    **GenusValidator (composed):**
    - All layers pass -> (True, [])
    - Any layer returns errors -> (False, [all errors from all layers])
    - Warnings from layers B/C do NOT cause overall failure (only items not prefixed with "warning:" count)
    - Constructor accepts list[ValidationLayer] for pluggability
  </behavior>
  <implementation>
    1. Create `src/pipeline/validation/__init__.py` exporting GenusValidator and all layer classes.

    2. In `src/pipeline/validation/layers.py`:
       - Define `ValidationLayer` Protocol with `validate(episode: dict) -> tuple[bool, list[str]]` method
       - `SchemaLayer`: takes EpisodeValidator in __init__, delegates validate()
       - `EvidenceGroundingLayer`: stateless, checks mode-specific evidence heuristics
         - Implement mode: check orchestrator_action.scope.paths is non-empty
         - Verify mode: check outcome has executor_effects or test-related content
         - Integrate mode: check outcome has git_events
         - All checks produce "warning:evidence:" prefixed messages, never hard fail
       - `NonContradictionLayer`: stateless, checks three design doc rules
         - Rule 1: mode=Explore with write_allowed in gates list -> warning
         - Rule 2: gate contains "no_network" and instruction contains network-related terms -> warning
         - Rule 3: gate contains "no_write_before_plan" and mode=Implement -> warning
         - All checks produce "warning:contradiction:" prefixed messages
       - `ConstraintEnforcementLayer`: takes list[dict] constraints in __init__
         - For each constraint, check scope.paths overlap with episode scope.paths
         - Repo-wide constraints (empty paths) apply to all episodes
         - Severity mapping: forbidden -> error (False), requires_approval -> warning, warning -> warning
         - Detection hint matching: check episode text content against detection_hints patterns
       - `EpisodeIntegrityLayer`: stateless structural checks
         - episode_id present and non-empty
         - provenance.sources is non-empty list
         - timestamp is valid ISO string
         - reaction.confidence in [0, 1] if reaction exists
         - All failures are hard errors

    3. In `src/pipeline/validation/genus_validator.py`:
       - `GenusValidator.__init__(layers: list[ValidationLayer])` stores layers
       - `GenusValidator.validate(episode: dict) -> tuple[bool, list[str]]`:
         - Runs all layers, collects all messages
         - Separates warnings (prefixed "warning:") from errors
         - Returns (True, warnings) if no hard errors, (False, errors + warnings) if any hard errors
       - `GenusValidator.validate_batch(episodes: list[dict]) -> dict` -- batch convenience method
       - Class method `GenusValidator.default(episode_validator=None, constraints=None)` creates instance with all 5 default layers

    4. Tests cover each layer independently plus the composed validator.
       Use minimal episode dict fixtures -- just enough fields for each layer's checks.
       Test the warning vs error distinction explicitly.
  </implementation>
</feature>
