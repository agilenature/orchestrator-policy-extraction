Add a helper function `_query_docs_cross_project()`:
```python
def _query_docs_cross_project(
    query: str,
    remote_db_path: str,
    top_n: int = 3,
) -> list[dict[str, Any]]:
    """Query doc_index on a remote project's DuckDB via ATTACH (READ_ONLY).

    Returns results in the same format as query_docs() with source='docs'.
    """
```
Logic:
1. Check `Path(remote_db_path).exists()` -- return [] if not
2. `conn = duckdb.connect(":memory:")`
3. `conn.execute(f"ATTACH '{remote_db_path}' AS remote (READ_ONLY)")`
4. Check if remote has doc_index: `SELECT name FROM duckdb_tables() WHERE database_name = 'remote' AND name = 'doc_index'`
5. If no doc_index, clean up (DETACH + close) and return []
6. For the query, reuse the axis-matching logic from `doc_query.py` but querying `remote.doc_index` instead of `doc_index`. Since `query_docs()` creates its own connection, we can't reuse it directly for ATTACH queries. Instead, inline a simplified version:
   - Tokenize query (import `_tokenize`, `_score_axis_match` from `src.pipeline.doc_query`)
   - Load axes from `remote.doc_index`
   - Match axes against query tokens
   - Query `remote.doc_index` for matching docs
   - Return results with `source: "docs"` and `match_reason: "direct"/"neighbor"`
   Note: Skip axis_edges expansion for cross-project queries (the remote DB may not have axis_edges). Direct axis matching only.
7. Clean up: `conn.execute("DETACH remote"); conn.close()` -- no `USE memory` needed since we never called `USE remote` (we used fully-qualified `remote.doc_index` names)
8. Entire function wrapped in try/except returning [] (fail-open)

Update the `query_cmd()` function's `--project` handling:

   a. Call `_resolve_project(project)` to get project config
   b. If project not found: `click.echo(f"[OPE Query] Unknown project: {project}")` and return

      - Call `_get_project_session_ids(project_config)` to get session IDs

   e. For `--source code`: print `click.echo(f"[OPE Query] Code search not available for remote projects.")` (code search is local-only per research Pitfall 8)
