## Decisions Made
- **delta_retrieval uses 'validated' status:** The plan specified querying `verdict = 'accepted'` but the memory_candidates schema uses `status IN ('pending', 'validated', 'suspended', 'rejected', 'split_required')`. Used `status = 'validated'` to match the actual schema constraint. [Rule 1 - Bug: fixed query to match actual schema]
- **specification_closure JOIN uses m.id instead of m.candidate_id:** The plan pseudocode used `m.candidate_id IS NULL` but the actual schema uses `id` as the primary key column name. Fixed to match actual DDL.
- **Harness is read-only on identification_reviews:** The harness never writes to or modifies the reviews table -- it writes only coverage snapshots for its own monotonicity tracking.
