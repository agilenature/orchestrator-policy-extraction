## Milestone Summary

**Decimal Phases:**
- Phase 13.1: Cross-Domain Axis Extraction (inserted after Phase 13 for cross-project CCD extraction)
- Phase 13.2: Causal Chain Analysis (inserted after 13.1 for decision-boundary analysis)
- Phase 13.3: Identification Transparency Layer (inserted after 13.2 for bootstrap-circularity resolution)
- Phase 14.1: Premise Registry + PAG (inserted after Phase 14 for CLAUDE.md protocol implementation)
- Phase 16.1: Topological Edge-Generation (inserted after Phase 16 for axis_edges causal map)

**Key Decisions:**
- Unix socket + Starlette for governance bus (locked at p99=1.6ms after spike)
- TENTATIVE_END / CONFIRMED_END state machine (X_ASK never an end trigger — Phase 14 locked decision)
- Two-tier fidelity: event_level signals fire immediately, episode_level signals deferred to CONFIRMED_END
- Bus fail-open: PAG and session_start must not block sessions if bus unavailable
- Behavioral parity as adoption gate for ReactiveX (not test coverage)
- ops.concat_map not flat_map for ordered stateful processors
- CCD (ccd_axis | scope_rule | flood_example) format for MEMORY.md — axis retrieval vs surface similarity
- Deposit-not-detect as governing axis: only components that deposit to memory_candidates are load-bearing

**Issues Resolved:**
- Bootstrap circularity in MEMORY.md validation (resolved via Harness — out-of-band invariant enforcer with no AI session state)
- Context overflow risk mitigated by CCD-format filing vs raw chunk accumulation
- Temporal closure dependency: real-time vs post-hoc boundary detection resolved by boundary_dependency classification
- Run-ID boundary dissolution: sessions across repos can share causal chain via OPE_RUN_ID
- X_ASK false-positive episode splits in post-hoc segmenter (fixed — X_ASK is structurally mid-episode)

**Issues Deferred:**
- OpenClaw Control Plane installation (builder/operator structural separation documented but not enforced)
- DAgger correction loop (ADV-03) — requires production shadow mode data
- Preference model training (PREF-01 through PREF-03) — requires episode volume validation first

**Technical Debt:**
- REQUIREMENTS.md traceability table had "Pending" for requirements completed after Phase 13 (all requirements now validated — archived in v1.0-REQUIREMENTS.md)
- Some CLIs use file-based DuckDB paths that assume `data/ope.db` exists (bus-absent sessions need explicit db path)
- Phase 26 was skipped (planned but removed from roadmap before execution)
