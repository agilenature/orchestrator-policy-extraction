**intelligence_profile.py** -- Per DDF-04: Basic IntelligenceProfile per-human aggregate.

`compute_intelligence_profile(conn, human_id: str) -> IntelligenceProfile | None`:
- SQL query against flame_events WHERE subject='human' AND human_id = ?:
  ```sql
  SELECT
      human_id,
      COUNT(*) AS flame_frequency,
      AVG(marker_level) AS avg_marker_level,
      MAX(marker_level) AS max_marker_level,
      SUM(CASE WHEN marker_level >= 6 THEN 1 ELSE 0 END)::FLOAT / NULLIF(COUNT(*), 0) AS flood_rate,
      COUNT(DISTINCT session_id) AS session_count
  FROM flame_events
  WHERE subject = 'human'
    AND human_id = ?
  GROUP BY human_id
  ```
- For spiral_depth: separate query using window function:
  ```sql
  WITH ordered AS (
      SELECT
          marker_level,
          ROW_NUMBER() OVER (PARTITION BY session_id ORDER BY created_at) AS rn,
          LAG(marker_level) OVER (PARTITION BY session_id ORDER BY created_at) AS prev_level
      FROM flame_events
      WHERE subject = 'human' AND human_id = ?
  ),
  streaks AS (
      SELECT
          marker_level,
          CASE WHEN prev_level IS NOT NULL AND marker_level > prev_level THEN 1 ELSE 0 END AS ascending
      FROM ordered
  )
  SELECT COALESCE(MAX(streak_len), 0) AS spiral_depth
  FROM (
      SELECT COUNT(*) AS streak_len
      FROM (
          SELECT ascending,
              SUM(CASE WHEN ascending = 0 THEN 1 ELSE 0 END) OVER (ORDER BY ROWID) AS grp
          FROM streaks
      )
      WHERE ascending = 1
      GROUP BY grp
  )
  ```
  NOTE: The spiral_depth computation is complex. Simplify: iterate flame_events in Python, track longest ascending streak. This avoids complex SQL window functions and is clearer.

- If no flame_events for this human_id, return None
- Return IntelligenceProfile with subject='human'

`compute_ai_profile(conn) -> IntelligenceProfile | None`:
- Same aggregation but WHERE subject='ai'
- human_id set to 'ai' in the returned profile
- Return IntelligenceProfile with subject='ai'

`compute_spiral_depth_for_human(conn, human_id: str) -> int`:
- Python-side computation:
  - Fetch all flame_events for human_id ordered by session_id, created_at
  - For each session: track longest ascending streak of marker_levels
  - Return max across all sessions
- This is used by both compute_intelligence_profile and the CLI

`list_available_humans(conn) -> list[str]`:
- SELECT DISTINCT human_id FROM flame_events WHERE subject='human' AND human_id IS NOT NULL
- Used by CLI to list available human IDs
