<feature>
  <name>PolicyViolationChecker</name>
  <files>src/pipeline/feedback/checker.py, tests/test_policy_violation_checker.py</files>
  <behavior>
    Pre-surfacing check: given a recommendation text, determines whether it should be suppressed based on active constraint detection_hints.

    Cases:
    - recommendation text contains forbidden constraint's detection_hint -> (True, constraint)
    - recommendation text contains requires_approval constraint's detection_hint -> (True, constraint)
    - recommendation text contains warning constraint's detection_hint -> (False, constraint)  [log only]
    - recommendation text matches no constraint hints -> (False, None)
    - constraint has no detection_hints -> skip (not matched)
    - case-insensitive matching: "DELETE" matches "delete" hint -> (True, constraint)

    Recommendation text composition: `reasoning + " " + " ".join(recommended_scope_paths) + " " + recommended_mode + " " + " ".join(recommended_gates)` -- concatenated from Recommendation model fields to maximize hint matching surface.
  </behavior>
  <implementation>
    Follow AmnesiaDetector._build_hint_patterns() pattern from src/pipeline/durability/evaluator.py:
    1. __init__(constraint_store) -- loads active constraints, pre-compiles detection_hints as re.compile(re.escape(hint), re.IGNORECASE)
    2. check(recommendation_text) -> tuple[bool, dict | None] -- iterate compiled patterns, return (should_suppress, matching_constraint)
    3. build_recommendation_text(recommendation) -> str -- class method that concatenates Recommendation fields into searchable text

    Pre-compile patterns ONCE at init time (not per-check). Filter to constraints that have detection_hints (skip empty). Include ALL active constraints regardless of severity for the pattern compilation, but only suppress for forbidden/requires_approval.
  </implementation>
</feature>

<feature>
  <name>PolicyFeedbackExtractor</name>
  <files>src/pipeline/feedback/extractor.py, src/pipeline/constraint_store.py, tests/test_policy_feedback_extractor.py</files>
  <behavior>
    Extracts policy_feedback constraints from shadow recommendations that were surfaced but blocked/corrected by human.

    Cases:
    - recommendation surfaced + reaction="block" -> new constraint with severity="forbidden", source="policy_feedback", status="candidate"
    - recommendation surfaced + reaction="correct" -> new constraint with severity="requires_approval", source="policy_feedback", status="candidate"
    - recommendation surfaced + reaction="approve" -> None (no constraint needed)
    - recommendation surfaced + reaction=None -> None
    - matching human constraint exists (2+ shared detection_hints keywords) -> enrich existing, return None
    - constraint ID uses SHA-256(text.lower() + JSON.dumps(sorted(scope_paths)) + ":policy_feedback")[:16]

    Promotion:
    - promote_confirmed(constraint_store, conn) -- query policy_error_events for candidate constraints with 3+ distinct session_ids having error_type='surfaced_and_blocked', promote matching constraints from candidate to active via constraint_store.add_status_history_entry()
  </behavior>
  <implementation>
    Follow EscalationConstraintGenerator pattern from src/pipeline/escalation/constraint_gen.py:
    1. extract(recommendation, episode_dict, constraint_store) -> dict | None
       - Check reaction_label is block/correct, else return None
       - Build constraint text from recommendation reasoning
       - Build scope_paths from recommendation.recommended_scope_paths
       - Build detection_hints from recommendation fields (mode, scope_paths, key terms from reasoning)
       - Check for matching human constraint via constraint_store.find_by_hints() (2+ shared keywords)
       - If match: enrich existing constraint examples, return None
       - If no match: build full constraint dict with source="policy_feedback", status="candidate"
       - Use SHA-256(text.lower().strip() + ":" + json.dumps(sorted(scope_paths)) + ":policy_feedback")[:16] for ID

    2. promote_confirmed(constraint_store, conn, min_sessions=3) -> int
       - Query DuckDB: SELECT constraint_id, COUNT(DISTINCT session_id) FROM policy_error_events WHERE error_type='surfaced_and_blocked' GROUP BY constraint_id HAVING COUNT(DISTINCT session_id) >= min_sessions
       - For each qualifying constraint_id, check if it exists in constraint_store as candidate
       - If candidate: call constraint_store.add_status_history_entry(cid, "active", now_utc) and update constraint["status"] = "active"
       - Return count of promoted constraints

    3. Add find_by_hints(detection_hints, min_overlap=2) -> dict | None to ConstraintStore:
       - Iterate all constraints, count shared keywords between given hints and constraint's detection_hints
       - Return first constraint with >= min_overlap shared hints (case-insensitive comparison)
       - If none found, return None
       - This method is used for dedup: checking if an equivalent human_correction constraint already exists

    Stateless extractor (same pattern as EscalationConstraintGenerator). Caller handles ConstraintStore.add() in pipeline integration (Plan 03).
  </implementation>
</feature>

    **src/pipeline/feedback/checker.py** -- PolicyViolationChecker:
    - `__init__(self, constraint_store: ConstraintStore)`: Load active constraints via `constraint_store.get_active_constraints()`. Pre-compile detection_hints patterns using `re.compile(re.escape(hint), re.IGNORECASE)` for each constraint that has non-empty detection_hints. Store as `list[tuple[dict, list[tuple[str, re.Pattern]]]]` -- same structure as research pattern.
    - `check(self, recommendation_text: str) -> tuple[bool, dict | None]`: Iterate compiled patterns. On first match: if severity in ("forbidden", "requires_approval") return (True, constraint). If severity == "warning" return (False, constraint). If no match: return (False, None).
    - `@staticmethod build_recommendation_text(recommendation) -> str`: Concatenate `recommendation.reasoning + " " + " ".join(recommendation.recommended_scope_paths) + " " + recommendation.recommended_mode + " " + " ".join(recommendation.recommended_gates)`. Handle None/empty fields gracefully.

    **src/pipeline/feedback/extractor.py** -- PolicyFeedbackExtractor:
    - `extract(self, recommendation, episode: dict, constraint_store: ConstraintStore) -> dict | None`:
      1. Check `episode.get("reaction_label")` in ("block", "correct"), else return None
      2. Build constraint text: `f"Policy recommended {recommendation.recommended_mode} mode but was {episode['reaction_label']}ed: {recommendation.reasoning}"` (truncate reasoning to 200 chars)
      3. scope_paths = recommendation.recommended_scope_paths or []
      4. Build detection_hints: extract key terms from recommendation (recommended_mode, scope_paths entries, first 3 words of reasoning > 3 chars)
      5. Dedup check: `constraint_store.find_by_hints(detection_hints, min_overlap=2)`. If found, return None (caller can enrich)
      6. Build constraint ID: `_make_constraint_id(text, scope_paths)`
      7. severity = "forbidden" if reaction == "block" else "requires_approval"
      8. Return full constraint dict with all required fields per constraint.schema.json
    - `_make_constraint_id(text, scope_paths) -> str`: `SHA-256(text.lower().strip() + ":" + json.dumps(sorted(scope_paths)) + ":policy_feedback")[:16]`. Per locked decision: this uses JSON.dumps (not "|") and always appends ":policy_feedback".
    - `_build_detection_hints(recommendation) -> list[str]`: Extract mode, scope paths, and key terms from reasoning.
    - `promote_confirmed(self, constraint_store: ConstraintStore, conn, min_sessions: int = 3) -> int`:
      1. Query: `SELECT constraint_id, COUNT(DISTINCT session_id) as cnt FROM policy_error_events WHERE error_type='surfaced_and_blocked' GROUP BY constraint_id HAVING cnt >= ?`
      2. For each result, find constraint in store. If status == "candidate": update status to "active", add status_history entry.
      3. Return count promoted.
    - `find_matching_human_constraint(self, constraint_store, detection_hints) -> dict | None`: Wrapper around `constraint_store.find_by_hints()` that filters to source != "policy_feedback" (only match against human/escalation constraints).

    **src/pipeline/constraint_store.py** -- Add find_by_hints() method:
    - `find_by_hints(self, detection_hints: list[str], min_overlap: int = 2) -> dict | None`:
      - Normalize input hints to lowercase set
      - For each constraint in self._constraints:
        - Get constraint's detection_hints, normalize to lowercase set
        - Count intersection size
        - If intersection >= min_overlap: return that constraint
      - Return None if no match
    - Place after `get_active_constraints()` method.
