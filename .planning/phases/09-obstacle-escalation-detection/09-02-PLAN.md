  <behavior>
The EscalationDetector walks a list of TaggedEvent objects and detects escalation sequences:

**Algorithm:**
1. Walk events in timestamp order
2. On O_GATE (negative/reject outcome) or O_CORR: open escalation window
3. Within window, count only NON-EXEMPT events toward the window_turns limit
4. Exempt tools (Read, Glob, Grep, WebFetch, WebSearch, Task): skip entirely, do not decrement window counter
5. X_ASK or X_PROPOSE: close/reset ALL pending windows (approval sought)
6. Bypass-eligible tool within window: emit EscalationCandidate, consume oldest matching window
7. Window expiry (window_turns non-exempt events passed without bypass): discard window

**Bypass eligibility (two-layer -- Pitfall 5):**
- Layer 1 (tag-based): T_RISKY, T_GIT_COMMIT, T_TEST tags on the event
- Layer 2 (tool-name-based): Write, Edit, Bash tool calls (even without T_RISKY tag)
- Either layer triggering = bypass detected
- Additionally: always_bypass_patterns in Bash command text (rm, chmod, sudo, etc.)

**Deduplication (Pitfall 2):**
- When a bypass event matches multiple pending windows (sequential blocks), consume only the OLDEST window
- Return at most 1 EscalationCandidate per bypass event

**Window counting (Pitfall 1):**
- Exempt tool events are completely transparent to the window -- they do not decrement the counter
- Only non-exempt events decrement the turn counter
- A 5-turn window means 5 non-exempt events, not 5 total events

**Test cases (30 total, Q7 locked decision):**

Positive (15 -- O_ESC expected):
- Blatant bypass (5): O_GATE -> immediate Bash/Write/Edit within 1-2 turns
- Delayed bypass within window (5): O_GATE -> 2-4 non-exempt events -> bypass on turn 4-5
- Indirect bypass via different tool (5): O_CORR -> bypass via Edit (not Bash), Bash rm, Write after rejection

Negative (15 -- no O_ESC expected):
- Read-only post-rejection (5): O_GATE -> only Read/Glob/Grep -> no bypass
- X_ASK resets window (5): O_GATE -> X_ASK -> Bash (window was reset)
- Window expired (5): O_GATE -> 6+ non-exempt events -> then Bash (too late)
