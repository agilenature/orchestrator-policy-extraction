**Create `src/pipeline/ddf/structural/detectors.py`**: Four detector functions + orchestrator.

CRITICAL RESEARCH FINDING: ALL axis queries MUST use `COALESCE(ccd_axis, axis_identified)` because flame_events has BOTH fields -- ccd_axis was added in Phase 17 via ALTER TABLE, while axis_identified has been the axis field since Phase 15. Queries checking only one field will silently miss matches.

CRITICAL: Use `COALESCE(prompt_number, 0)` for prompt_number in Gravity Check queries because some flame_events have NULL prompt_number.

1. **`detect_gravity_checks(conn, session_id, subject, gravity_window=3, assessment_session_id=None) -> list[StructuralEvent]`**:
   - Query flame_events for all L5+ events with non-null axis (COALESCE(ccd_axis, axis_identified) IS NOT NULL).
   - For each L5+ event, LEFT JOIN to find any L0-L2 flame_events for the SAME axis, SAME session, SAME subject, within +-gravity_window prompts (use ABS(COALESCE(fe_low.prompt_number,0) - COALESCE(fe_high.prompt_number,0)) <= gravity_window).
   - If assessment_session_id filtering needed, add `AND (fe.assessment_session_id IS NULL)` for production sessions.
   - signal_passed=True when grounding found (COUNT(low) > 0); signal_passed=False when absent.
   - structural_role='grounding' when passed, None when failed.
   - contributing_flame_event_ids = [high_event_id] + list of matching low event IDs.
   - Use StructuralEvent.make_id(session_id, prompt_number, 'gravity_check') for deterministic IDs.

2. **`detect_main_cables(conn, session_id, subject, assessment_session_id=None) -> list[StructuralEvent]`**:
   - Query flame_events for L5+ events with non-null axis.
   - For each axis, count DISTINCT scope_path prefixes from the flame_events in that session. A scope prefix is derived from evidence_excerpt or source_episode_id. Simpler approach per research: count distinct episodes (source_episode_id) OR count distinct session-level axis occurrences. BEST approach: check if the axis appears in `axis_edges` (which implies cross-domain connectivity and generalization_radius >= 2) OR count distinct flame_events with different source_episode_id values for that axis in the session.
   - SIMPLEST correct approach: For each L5+ flame_event axis, query `SELECT COUNT(DISTINCT source_episode_id) FROM flame_events WHERE session_id=? AND subject=? AND COALESCE(ccd_axis, axis_identified) = ? AND source_episode_id IS NOT NULL AND (assessment_session_id IS NULL)`. If count >= 2, generalization_radius >= 2, main cable signal passes. If source_episode_id is NULL for all, fall back to checking axis_edges: `SELECT COUNT(*) FROM axis_edges WHERE (axis_a = ? OR axis_b = ?) AND status = 'active'`. If >= 1, the axis has cross-domain connectivity, signal passes.
   - signal_passed=True when generalization_radius >= 2 (grounded main cable); False when < 2 (floating cable).
   - structural_role='load_bearing' when passed, None when failed.
   - op8_status: set to 'pass' when signal_passed=True, 'fail' when signal_passed=False AND subject='ai', 'na' for human subjects.

3. **`detect_dependency_sequencing(conn, session_id, subject, assessment_session_id=None) -> list[StructuralEvent]`**:
   - For each L5+ flame_event introducing a new axis in the session (first occurrence of that COALESCE(ccd_axis, axis_identified) in session flame_events ordered by prompt_number):
   - Check axis_edges: does this axis have prerequisite axes? Query: `SELECT axis_a FROM axis_edges WHERE axis_b = ? AND status = 'active' UNION SELECT axis_b FROM axis_edges WHERE axis_a = ? AND status = 'active'` to find connected axes.
   - If prerequisite axes exist: check whether at least one prerequisite axis has a prior L3+ flame_event in the session (earlier prompt_number). If yes: signal_passed=True (dependency respected). If no: signal_passed=False.
   - If NO prerequisite axes exist in axis_edges: skip this event (do not emit a structural event -- no violations detectable). This handles the empty axis_edges case gracefully per Pitfall 6.
   - structural_role='hierarchical'.

4. **`detect_spiral_reinforcement(conn, session_id, subject, assessment_session_id=None) -> list[StructuralEvent]`**:
   - Cross-reference: query project_wisdom for entries whose metadata or source links match the current session. Specifically: `SELECT wisdom_id, title FROM project_wisdom WHERE metadata LIKE '%session_id%'` where session_id is the current session. OR simpler: check if any spiral promotions happened for this session by querying flame_events for L6+ events that have deposited_to_candidates=True.
   - SIMPLEST approach: query flame_events for events in this session with marker_level >= 6 AND flood_confirmed = True AND subject matches. Each such event represents a confirmed flood (spiral reinforcement candidate). Count them.
   - Emit one StructuralEvent per confirmed L6+ flood with signal_passed=True, signal_type='spiral_reinforcement', structural_role='reinforcing'.
   - contributing_flame_event_ids = [the flood event ID].

5. **`detect_structural_signals(conn, session_id, config=None) -> list[StructuralEvent]`**: Orchestrator function.
   - Load StructuralConfig from config or use defaults.
   - For each subject in ('human', 'ai'): call all four detectors, concatenate results.
   - Return combined list of all StructuralEvents.

**Create `src/pipeline/ddf/structural/computer.py`**: Score computation.

**`compute_structural_integrity(conn, session_id, subject, config=None) -> StructuralIntegrityResult | None`**:
   - First check: count L5+ flame_events for this session+subject. If 0: return None (locked: Score=NULL when no L5+ events).
   - Query structural_events for this session+subject, grouped by signal_type.
   - gravity_ratio: gravity_check pass_count / max(1, total_L5plus_flame_events). If total_L5plus is 0: use 0.5 (neutral_fallback from config).
   - main_cable_ratio: main_cable pass_count / max(1, main_cable_total). If main_cable_total is 0: use 0.5 (neutral_fallback).
   - dependency_ratio: dependency_pass_count / max(1, dependency_total). If dependency_total is 0: use 0.5 (neutral_fallback).
   - spiral_capped: min(spiral_reinforcement_count, 3) / 3.0. This is a bonus, NOT ratio-based with neutral fallback. 0 spirals = 0.0 contribution.
   - Score = 0.30 * gravity_ratio + 0.40 * main_cable_ratio + 0.20 * dependency_ratio + 0.10 * spiral_capped
   - Return StructuralIntegrityResult with all fields populated.

**Create `src/pipeline/ddf/structural/op8.py`**: Op-8 correction deposit.

**`deposit_op8_corrections(conn, session_id, config=None) -> int`**:
   - Query ai_flame_events (subject='ai') at marker_level >= 5 where BOTH conditions are true:
     (a) flood_confirmed = False
     (b) No L0-L2 ai_flame_event for the same COALESCE(ccd_axis, axis_identified) within the same session.
   - The check for condition (b) is the SAME Gravity Check logic but specifically for AI subject and session-wide (not windowed).
   - Group floating events by axis (COALESCE(ccd_axis, axis_identified)). One correction candidate per floating axis per session.
   - For each floating axis:
     - candidate_id = SHA-256("op8:{session_id}:{ccd_axis}")[:16] -- LOCKED dedup ID format.
     - scope_rule = "This axis appeared in AI reasoning without concrete grounding (no L0-L2 co-occurrence). Require Gravity Check evidence before asserting."
     - flood_example = verbatim evidence_excerpt from the FIRST L5+ flame_event for that axis (the ungrounded assertion).
     - Direct INSERT OR REPLACE into memory_candidates with: id=candidate_id, ccd_axis=axis, scope_rule, flood_example, status='pending', source_type='op8_correction', fidelity=2, confidence=0.60, subject='ai', session_id, source_flame_event_id=(first event's ID), detection_count=(count of floating events for that axis), pipeline_component='op8_structural'.
   - Return count of corrections deposited.
