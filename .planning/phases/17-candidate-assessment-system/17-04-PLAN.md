Class `AssessmentReporter`:
```python
class AssessmentReporter:
    """Generates Assessment Reports and deposits them as terminal memory candidates.

    The Assessment Report is the terminal deposit of Phase 17:
    every FlameEvent, TE score, and rejection analysis is instrumental
    to producing this CCD-quality entry that upgrades the AI's filing system.

    The report is deposited to memory_candidates with:
    - source_type='simulation_review'
    - fidelity=3 (highest: forced DDF L5-7 reasoning)
    - confidence=0.85

    Args:
        conn: DuckDB connection.
    """

    def __init__(self, conn: duckdb.DuckDBPyConnection) -> None:
        self._conn = conn
```

Method `generate_report(self, session_id: str, scenario_id: str, candidate_id: str) -> AssessmentReport`:

1. **FlameEvent timeline**: Query flame_events for the session, ordered by prompt_number:
   ```sql
   SELECT flame_event_id, prompt_number, marker_level, marker_type,
          evidence_excerpt, quality_score, axis_identified, flood_confirmed,
          subject
   FROM flame_events
   WHERE session_id = ?
   ORDER BY prompt_number, subject
   ```

2. **Level distribution**: Count events per level (L0-L7) for human subject:
   ```python
   level_distribution = {}
   for event in human_events:
       key = f"L{event.marker_level}"
       level_distribution[key] = level_distribution.get(key, 0) + 1
   ```

3. **TE metrics** (3-metric formula): Query assessment_te_sessions:
   ```sql
   SELECT candidate_te, raven_depth, crow_efficiency, trunk_quality,
          candidate_ratio, scenario_baseline_te, fringe_drift_rate
   FROM assessment_te_sessions
   WHERE session_id = ? AND candidate_id = ?
   ```

4. **Percentile rank**: Query assessment_baselines for the scenario:
   ```sql
   SELECT n_assessments, mean_ratio, stddev_ratio
   FROM assessment_baselines
   WHERE scenario_id = ?
   ```
   If n_assessments >= 10 AND mean_ratio and stddev_ratio are not None:
   - Compute z-score: `z = (candidate_ratio - mean_ratio) / stddev_ratio`
   - Convert to percentile using normal CDF approximation (no scipy needed):
     ```python
     import math
     percentile = 0.5 * (1 + math.erf(z / math.sqrt(2)))
     ```
   If n_assessments < 10: percentile_rank = None (baseline_pending)

5. **Axis quality scores**: For each unique axis_identified in human flame_events:
   ```python
   axis_quality_scores = {}
   for axis in unique_axes:
       events = [e for e in human_events if e.axis_identified == axis]
       avg_quality = sum(e.quality_score or 0 for e in events) / len(events)
       axis_quality_scores[axis] = round(avg_quality, 3)
   ```

6. **Flood rate**: Count events with flood_confirmed=True / total human events.

7. **Spiral evidence**: Query for ascending marker_level sequences in the session:
   ```python
   spiral_evidence = []
   levels = [e.marker_level for e in sorted(human_events, key=lambda x: x.prompt_number)]
   # Detect ascending runs of length >= 3
   current_run = [levels[0]] if levels else []
   for lvl in levels[1:]:
       if lvl > current_run[-1]:
           current_run.append(lvl)
       else:
           if len(current_run) >= 3:
               spiral_evidence.append(f"L{current_run[0]}->L{current_run[-1]} ({len(current_run)} steps)")
           current_run = [lvl]
   if len(current_run) >= 3:
       spiral_evidence.append(f"L{current_run[0]}->L{current_run[-1]} ({len(current_run)} steps)")
   ```

8. **AI contribution profile**: Query ai_flame_events for the session:
   ```sql
   SELECT COUNT(*), AVG(marker_level)
   FROM flame_events
   WHERE session_id = ? AND subject = 'ai'
   ```

9. **Rejection analysis**: Query from rejection detector results or re-compute:
   Use `RejectionDetector.detect_rejections()` from Plan 03, then count by type.

10. Build and return `AssessmentReport` model with all fields populated.

Method `deposit_report(self, report: AssessmentReport) -> str | None`:

The terminal deposit. Writes the Assessment Report as a CCD-quality memory_candidates entry.

1. Build CCD-format fields:
   ```python
   ccd_axis = f"assessment-{report.scenario_id[:8]}-{report.candidate_id[:8]}"
   scope_rule = (
       f"Assessment session measuring candidate {report.candidate_id} "
       f"on scenario {report.scenario_id} at DDF target level "
       f"{report.level_distribution}. Candidate TE ratio: "
       f"{report.candidate_ratio}. L5 rejections: {report.rejections_level5}."
   )
   flood_example = (
       f"Session {report.session_id}: {report.flame_event_count} flame events, "
       f"level distribution {report.level_distribution}, "
       f"candidate TE {report.candidate_te:.4f}, "
       f"AI contribution at avg L{report.ai_avg_marker_level:.1f}."
   )
   ```

2. Deposit using existing deposit function with assessment-specific fields:
   ```python
   # Cannot use deposit_to_memory_candidates directly because it doesn't support
   # source_type, fidelity, confidence. Use direct INSERT.
   candidate_id_hash = hashlib.sha256(
       f"assessment:{report.session_id}".encode()
   ).hexdigest()[:16]

   conn.execute(
       """
       INSERT OR REPLACE INTO memory_candidates (
           id, source_instance_id, ccd_axis, scope_rule, flood_example,
           pipeline_component, status, source_flame_event_id, fidelity,
           detection_count, confidence, subject, session_id, source_type
       ) VALUES (?, ?, ?, ?, ?, ?, 'pending', NULL, ?, 1, ?, ?, ?, ?)
       """,
       [
           candidate_id_hash,
           report.session_id,
           ccd_axis,
           scope_rule,
           flood_example,
           "assessment_reporter",
           report.fidelity,  # 3
           report.confidence,  # 0.85
           "assessment",  # subject
           report.session_id,
           report.source_type,  # 'simulation_review'
       ],
   )
   ```

3. Return candidate_id_hash.

Method `check_auto_calibration(self, scenario_id: str) -> str | None`:

Auto-calibration check per locked decision Q7:
1. Query assessment_baselines for the scenario
2. If n_assessments < 10: return None (insufficient data)
3. Query last 3 consecutive assessment ratios:
   ```sql
   SELECT candidate_ratio
   FROM assessment_te_sessions
   WHERE scenario_id = ?
   ORDER BY assessment_date DESC
   LIMIT 3
   ```
4. If all 3 ratios > 1.3: propose downgrade (scenario too easy)
   If all 3 ratios < 0.5: propose upgrade (scenario too hard)
5. Write proposal to memory_candidates (NOT auto-update ddf_target_level):
   ```python
   ccd_axis = f"auto-calibration-{scenario_id[:8]}"
   scope_rule = (
       f"Scenario {scenario_id} may need recalibration: "
       f"last 3 ratios were {ratios}. "
       f"{'Scenario too easy - consider increasing DDF level' if too_easy else 'Scenario too hard - consider decreasing DDF level'}."
   )
   # Deposit as simulation_review for human review
   ```
6. Return candidate_id if proposal deposited, None otherwise.

<pitfalls>
1. **Terminal deposit uses direct INSERT, not deposit_to_memory_candidates().** The existing deposit function doesn't support source_type, fidelity, or confidence fields. Use direct INSERT OR REPLACE with all required columns. Ensure ccd_axis, scope_rule, flood_example are all non-empty (memory_candidates has CHECK constraints on these).
2. **3-metric TE only.** Assessment TE is `raven_depth * crow_efficiency * trunk_quality`. Do NOT include transport_speed. Verify this in tests by checking that the computation matches the 3-factor product, not 4.
3. **Auto-calibration does NOT auto-update.** The proposal is written to memory_candidates as source_type='simulation_review' for human review via `intelligence memory-review`. It NEVER directly modifies project_wisdom.ddf_target_level.
4. **Production IntelligenceProfile exclusion.** The existing `compute_intelligence_profile()` in `src/pipeline/ddf/intelligence_profile.py` queries flame_events without filtering assessment_session_id.
