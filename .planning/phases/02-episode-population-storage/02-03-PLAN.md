<feature>
  <name>ReactionLabeler</name>
  <files>src/pipeline/reaction_labeler.py, tests/test_reaction_labeler.py</files>
  <behavior>
    The ReactionLabeler takes the next human message after an episode boundary and classifies it as a reaction.

    **Inputs:**
    - `next_human_message: dict | None` -- The next human_orchestrator event after episode end. None if end of session.
    - `episode_end_trigger: str | None` -- The tag that ended the episode (T_TEST, X_PROPOSE, etc.)
    - `episode_outcome: str | None` -- The segment outcome (success, failure, timeout, etc.)

    **Output:** `dict | None` -- `{label, message, confidence}` or None if no reaction determinable.

    **Two-tier pattern matching:**
    - Strong patterns: Higher confidence (0.85 for strong match)
    - Weak patterns: Lower confidence (0.55 for weak match)

    **Priority ordering (checked in this order, first match wins):**
    1. `block`: Strong: `^NO\b`, `\bstop\b`, `\bnever\b`, `\bdon't\s+do\s+that\b`. Weak: `\bdon't\b`, `\bavoid\b`
    2. `correct`: Strong: `\b(no|nope),?\s+(do|use|try|change)`, `\bthat's?\s+(wrong|not|incorrect)\b`, `\bchange\s+it\s+to\b`. Weak: `\bfix\b`, `\binstead\b`
    3. `redirect`: Strong: `\binstead\s+focus\b`, `\bdifferent\s+direction\b`, `\bswitch\s+to\b`, `\bfirst\s+(do|handle|fix)\b`. Weak: `\bbefore\s+that\b`, `\bpriority\b`
    4. `question`: Strong: `\bwhy\s*\?`, `\bwhat\s+about\b`, `\bhow\s+does\b`, `\bexplain\b`. Weak: `\?$`
    5. `approve`: Strong: `\b(yes|yeah|looks?\s+good|go\s+ahead|LGTM|approved?)\b`. Weak: `\b(ok|sure|fine|proceed|that\s+works)\b`

    **Special cases:**
    - If next message has O_CORR tag: return `{label: "correct", confidence: 0.9}` regardless of text
    - If next message has O_DIR/O_GATE tag without correction keywords: return `{label: "approve", confidence: 0.5}` (implicit approval)
    - If no next message (None): return None
    - If no patterns match and message is not a start trigger: return `{label: "unknown", confidence: 0.3}`

    **Test cases:**
    - "Looks good, go ahead" -> approve, confidence >= 0.7
    - "yes" -> approve, confidence >= 0.7
    - "ok" -> approve, confidence >= 0.5 (weak)
    - "No, do it differently" -> correct, confidence >= 0.7
    - "that's wrong" -> correct, confidence >= 0.7
    - "fix it" -> correct, confidence >= 0.5 (weak)
    - "STOP" -> block, confidence >= 0.7
    - "never do that" -> block, confidence >= 0.7
    - "don't" -> block, confidence >= 0.5 (weak)
    - "switch to the other approach" -> redirect, confidence >= 0.7
    - "first fix the tests" -> redirect, confidence >= 0.7
    - "why did you do that?" -> question, confidence >= 0.7
    - "what about error handling?" -> question, confidence >= 0.7
    - O_CORR tagged message -> correct, 0.9
    - O_DIR tagged message (no correction words) -> approve, 0.5
    - No next message -> None
    - Ambiguous text "hmm" -> unknown, 0.3
    - Priority test: "No, stop and switch to a different approach" -> block (not redirect), >= 0.7
  </behavior>
  <implementation>
    Create `ReactionLabeler` class with:
    - `__init__(self, config: PipelineConfig)` -- pre-compiles regex patterns from REACTION_PATTERNS
    - `label(self, next_human_message: dict | None, episode_end_trigger: str | None, episode_outcome: str | None) -> dict | None` -- main entry point
    - Private helpers:
      - `_classify_text(text: str) -> tuple[str | None, float]` -- keyword matching with priority
      - `_parse_payload(payload) -> dict` -- extract text from event payload (handles JSON string or dict)

    Pre-compile all regex patterns with `re.IGNORECASE` in `__init__` for performance. Store as list of `(label, tier, compiled_pattern)` tuples sorted by priority order.

    Pattern matching algorithm:
    1. For each label in priority order (block, correct, redirect, question, approve):
       a. Check strong patterns first
       b. If any strong pattern matches, return (label, 0.85)
       c. Check weak patterns
       d. If any weak pattern matches, record as candidate
    2. If candidates found, return highest-priority candidate with 0.55 confidence
    3. If no matches, return (None, 0.0)
  </implementation>
</feature>
