6. **test_level6_deposits_to_candidates** -- Create Level 6 flood_confirmed=true FlameEvent, call deposit_to_memory_candidates(), verify row in memory_candidates with status='pending'.

10. **test_dedup_warning_on_existing_axis** -- Pre-populate MEMORY.md with an entry containing ccd_axis text, insert candidate with same axis, simulate accept, verify warning message in CLI output.

**TestDDF08_FringeDrift:**
11. **test_fringe_drift_zero_when_concept_named** -- Insert marker_level 1 + marker_level 6 flood_confirmed in same session. compute_fringe_drift() returns 0.0.
12. **test_fringe_drift_one_when_drifted** -- Insert marker_level 2 but no L6+ in same session. Returns 1.0.
13. **test_fringe_drift_null_no_fringe** -- Insert only L4+ events. Returns None.
14. **test_fringe_drift_stored_on_te_row** -- After compute_te_for_session, verify fringe_drift_rate column populated.

**TestDDF09_TrunkQuality:**
15. **test_trunk_quality_pending_on_initial_compute** -- compute_te_for_session() sets trunk_quality=0.5, status='pending'.
16. **test_trunk_quality_backfill_with_3_sessions** -- Create 4 sessions: session_1 has L0 trunk event with axis_identified='test-axis', sessions 2-4 have L5+ events with same axis. Run backfill. session_1's trunk_quality should be > 0.0, status='confirmed'.
17. **test_trunk_quality_stays_pending_insufficient_sessions** -- Create only 2 sessions. Backfill runs but session_1 stays pending.
