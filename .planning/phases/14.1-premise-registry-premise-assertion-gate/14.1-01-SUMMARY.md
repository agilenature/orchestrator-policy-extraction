## Decisions Made
- **DuckDB expression indexes not supported:** RESEARCH.md Pattern 3 specified `CREATE INDEX ON premise_registry(json_extract_string(staining_record, '$.stained'))` but DuckDB raises `BinderException`. Removed expression index; stained premise queries use WHERE clause filtering instead. No performance impact (table size is small).
- **Lazy import of create_premise_schema:** Used local import inside create_schema() to avoid adding premise module to storage/schema.py's top-level imports. Keeps backward compatibility for existing tests that import storage/schema.py without the premise module on the import path.
- **Registry stub in Task 1:** Created minimal registry.py stub so __init__.py could import PremiseRegistry during Task 1 test collection. Full CRUD implementation followed in Task 2.
