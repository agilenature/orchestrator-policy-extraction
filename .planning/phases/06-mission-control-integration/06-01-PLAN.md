    - Import better-sqlite3 types (import type Database from 'better-sqlite3')

  <action>
    Create the Python-side bridge that reads Mission Control's SQLite database using DuckDB's SQLite extension.

    **src/pipeline/bridge/__init__.py:**
    - Export MCBridgeReader

    **src/pipeline/bridge/mc_reader.py:**
    - Class MCBridgeReader with __init__(mc_db_path: str, ope_conn: duckdb.DuckDBPyConnection)
    - Method attach(): Execute `ATTACH '{mc_db_path}' AS mc (TYPE sqlite)` -- handle the case where the extension needs to be installed/loaded first via `INSTALL sqlite; LOAD sqlite;` with try/except for already-installed
    - Method detach(): Execute `DETACH mc`
    - Context manager support (__enter__/__exit__) that calls attach/detach
    - Method list_episodes(status: str = 'completed', has_reaction: bool = True) -> list[dict]: Query mc.episodes with optional filters, parse JSON columns (observation, orchestrator_action, outcome, provenance, constraints_extracted, labels) via json.loads
    - Method import_episodes() -> list[Episode]: Read completed episodes with reactions from MC, validate each against the Pydantic Episode model, return valid episodes. Log warnings for any that fail validation (don't raise).
    - Method get_episode_events(episode_id: str) -> list[dict]: Query mc.episode_events for a given episode, parse JSON payload
    - Method get_constraints() -> list[dict]: Query mc.constraints, parse JSON arrays
    - Use short-lived queries (attach, query, detach pattern per research Pitfall 6 guidance) to avoid holding SQLite locks

    **tests/test_mc_bridge.py:**
    - Use a temporary SQLite database (created with Python's sqlite3 module) as the MC database
    - Test setup: Create the episode tables in the temp SQLite DB using the same DDL from the TypeScript schema (replicate the CREATE TABLE statements in the test fixture)
    - Test: Insert a sample episode directly into SQLite, then read it via MCBridgeReader.list_episodes() and verify all fields round-trip correctly
    - Test: Insert a sample episode with valid Pydantic-compatible data, call import_episodes(), verify it returns a valid Episode instance
    - Test: Insert episode_events, call get_episode_events(), verify events returned with parsed payloads
    - Test: Attach/detach context manager works cleanly
    - Test: Invalid episode data logs warning but doesn't raise
    - At least 8 tests covering the bridge operations
